## Task 2 — Add debounced `useEffect` for auto-recalculation

Insert the following `useEffect` directly after the closing `}` of the existing mount

```jsx
// ── Auto-recálculo em tempo real ──────────────────────────────
useEffect(() => {
  const PV = parseFloat(valorOriginal) || 0;
  if (!PV || !dataCalculo) {
    setResultado(null);
    return;
  }
  const timer = setTimeout(() => {
    calcularSilencioso();
  }, 350);
  return () => clearTimeout(timer);
}, [
  valorOriginal, dataCalculo, dataVencimento,
  indexador, jurosAM, multa, baseMulta,
  encargos, bonificacao,
  honorariosPct, incluirHonorarios,
  dividasSel, devId,

```

  calculation branches confirmed in research section 4.1).

- Do NOT wrap the function in `useCallback` — not needed for this use case.
- Insert point: immediately after line 3892 (`  }, []);`), before `async function handleAtualizarIndices`.

## Acceptance criteria

1. When `valorOriginal` or any parameter input changes, the result panel updates within ~350ms
   without any user clicking "Calcular".
2. When `valorOriginal` is empty or `dataCalculo` is empty, the result panel shows the empty
   state (not an error, not stale data).
3. Clicking "Recalcular" button still works — calls `calcular()` which shows toast on invalid
   input and records `logAudit` on success.
4. No toast fires during auto-recalculation (even when fields are temporarily empty during
   editing).
5. The audit log (`logAudit`) is only called via the manual button click path, never via the
   `useEffect` path.
6. No `onChange` handlers are modified — all inputs remain as-is.
7. No layout, color, or visual structure changes other than the two text strings in Task 3.
8. App compiles without errors: `cd src/mr-3/mr-cobrancas && npm run build` (or `npm start`)
   completes with no errors.
