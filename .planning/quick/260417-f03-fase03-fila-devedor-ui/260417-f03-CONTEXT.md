<decisions>
## Implementation Decisions

### Estrutura de componentes
- Arquivo separado: `src/components/FilaDevedor.jsx`
- Todas as 5 telas como funções internas (padrão do projeto)
- Default export: `FilaDevedor({ user, devedores, credores })`
- Sub-views controladas por `const [view, setView]` interno

### User → Operador ID bridge
- Ao montar FilaDevedor, buscar `operadores WHERE usuario_id = user.id AND ativo = true`
- Se não encontrado: criar novo operador com `{ usuario_id: user.id, ativo: true }`
- Armazenar UUID resultante em `const [operadorId, setOperadorId]`
- Mostrar loading enquanto resolve

### Acesso
- Visível para todos os usuários (sem restrição de role)

### listarFila() no filaDevedor.js
- Nova 8ª função pública para listagem enriquecida do Painel
- Batch query: fila_cobranca → contratos (batch) → devedores (batch)
- Máximo 3 queries independente do tamanho da fila
- Filtros suportados: equipe_id, prioridade, status_fila

### Painel (Tela 1)
- 4 cards de contador: AGUARDANDO, EM_ATENDIMENTO, ACIONADO+hoje, BLOQUEADO
- Filtros: equipe, prioridade, credor — client-side após carregar
- Tabela com PriorityBadge
- Botão "Atualizar Fila" → filaDevedor.entrarNaFila()

### Operador Lobby (Tela 2)
- Card centralizado com contador AGUARDANDO + badge de prioridade média
- Botão "Iniciar Atendimento" → filaDevedor.proximoDevedor(operadorId)
- Se retornar data null → toast "Fila vazia"

### Atendimento (Tela 3)
- Recebe dadosIniciais do proximoDevedor() (fila, devedor, contrato, parcelas, eventos)
- Modal de novo evento com campos condicionais por tipo_evento
- Botão "Próximo >" desabilitado até registrar evento
- Ao clicar Próximo → filaDevedor.proximoDevedor() → recarregar ou voltar ao lobby

### Pesquisa (Tela 4)
- Busca em devedores passados via prop
- Filtros simples client-side (nome, cpf_cnpj, contrato)
- Tabela com ações básicas

### Histórico (Tela 5)
- Query eventos_andamento com filtro de data
- Export CSV via download de string
- Filtros de data: 7d, 30d, personalizado

### App.jsx mudanças mínimas
- Adicionar icon `fila` no objeto I (SVG de lista/queue)
- Inserir `{ id: "fila", label: "Fila de Cobrança", icon: I.fila, color: "#f97316", bg: "rgba(249,115,22,.18)" }` entre regua e peticao
- Importar FilaDevedor
- Adicionar case "fila" no renderPage

</decisions>
