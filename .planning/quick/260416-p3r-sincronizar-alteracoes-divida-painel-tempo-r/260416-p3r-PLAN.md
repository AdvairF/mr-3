## Task Details

### Task 1 — Fix bloco "período final"

**Before:**
```js
saldo = pcFinal + jurosFinal;
```

**After:**
```js
const multaFinal = primeiroperiodo ? pcFinal * (multaPctDiv / 100) : 0;
const honorariosFinal = primeiroperiodo
  ? (pcFinal + jurosFinal + multaFinal) * (honorariosPctDiv / 100)
  : 0;
saldo = pcFinal + jurosFinal + multaFinal + honorariosFinal;
```

**Why:** A linha original ignora multa e honorários. O flag `primeiroperiodo` indica que
nenhum período de pagamento foi processado — portanto é a primeira vez que encargos
são aplicados sobre esta dívida. Mesma lógica já usada no `gerarPlanilhaPDF`.

## Commit Message
```
fix(260416-p3r): multa e honorários no saldo devedor sem pagamentos parciais

calcularSaldoDevedorAtualizado não aplicava multa/honorários quando não havia
pagamentos parciais (bloco "período final" só somava juros+correção). Editing
multa_pct ou honorarios_pct não alterava o valor exibido no painel.

Fix: aplicar multa/honorários no período final quando primeiroperiodo ainda é true.
```

## Threat Model

- **Regressão para devedores COM pagamentos:** `primeiroperiodo` só é true se nenhum
  período de pagamento foi processado — devedores com pagamentos não são afetados.
- **Dupla contagem:** Multa e honorários são aplicados uma única vez (só quando `primeiroperiodo`),
  igual ao comportamento de `gerarPlanilhaPDF`.
- **Saldo negativo:** `Math.max(0, saldo)` na linha 161 garante que saldo nunca fica negativo.
