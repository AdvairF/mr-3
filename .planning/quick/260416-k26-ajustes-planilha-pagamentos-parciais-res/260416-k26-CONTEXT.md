<decisions>
## Implementation Decisions

### Honorários
- Base de cálculo: sobre o **total atualizado** (valor original + correção + juros + multa)
- `honorariosVal = totalAtualizado * (div.honorarios_pct / 100)`
- Omitir linha se `honorarios_pct === 0`

### Múltiplas dívidas
- Distribuição: **sequencial** — pagamentos abate a 1ª dívida primeiro, depois a 2ª etc.
- Cada dívida tem sua própria seção com cabeçalho, tabela e subtotal
- Ao final: bloco TOTAL GERAL somando todos os saldos

### Contagem de meses para descrição de período
- `Math.floor(dias / 30)` onde dias = diferença entre datas em milissegundos / 86400000
- Exibir: "N meses" (se 0 meses, exibir "X dias" em vez disso)

### Resumo executivo (novo layout)
- Layout vertical em vez de horizontal (box mais alto)
- 8 linhas: Valor Original | (+) Multa | (+) Honorários | (+) Correção Monetária | (+) Juros | (=) Total Atualizado | (-) Total Pago | (=) SALDO DEVEDOR FINAL
- Omitir linhas zeradas (multa, honorários, correção) para manter clean

### Planilha detalhada (linhas separadas)
- Linha: Valor original (opening)
- Linha: Multa (X%) = R$ XX — só no 1º período, se > 0
- Linha: Honorários (X%) = R$ XX — só se > 0
- Linha: Juros X% a.m. — DD/MM a DD/MM (N meses) = R$ XX — se > 0
- Linha: Correção monetária (IDX) — DD/MM a DD/MM = R$ XX — se > 0
- Linha: Pagamento parcial (abatimento)
- Repetir juros/correção para cada período subsequente
- Linha final: SALDO DEVEDOR FINAL

### Claude's Discretion
- Altura exata do box do resumo executivo (adaptar ao número de linhas visíveis)
- Quebra de página automática entre seções de dívidas quando necessário
