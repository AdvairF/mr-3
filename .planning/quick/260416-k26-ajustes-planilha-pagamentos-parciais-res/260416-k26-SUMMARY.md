# Quick 260416-k26: Refatorar gerarPlanilhaPDF() — Planilha PDF completa

One-liner: Refatoração de gerarPlanilhaPDF() em App.jsx com resumo executivo vertical de 8 linhas, rows separados por componente (Multa/Honorários/Correção/Juros), seções por dívida e acumuladores globais de totais.

## O que foi alterado em gerarPlanilhaPDF()

### Bloco de cálculo iterativo (linhas ~1997–2122 originais)

**Antes:** Loop único sobre `pagamentos`, calculava correção+juros+multa agrupados em uma única row de acréscimo, sem separação por dívida, sem acumuladores de totais separados.

**Depois:**
- Introduzido `dividasCalc` (array de dívidas elegíveis) como unidade de iteração
- Loop externo `for (let di = 0; di < dividasCalc.length; di++)` processa cada dívida independentemente
- `pgtoRestantes` com campo `.remaining` para distribuição sequencial — cada pagamento abate dívidas em ordem sem risco de reutilização
- Rows separados por componente por período:
  - `isMulta` — apenas 1º período da dívida, se multaPct > 0
  - `isHonorarios` — apenas 1º período da dívida, se honorariosRowVal > 0.005
  - `isCorr` — cada período onde corrAbs > 0.005
  - `isJuros` — cada período onde juros > 0.005, label inclui datas e N meses/dias
- Acumuladores globais: `totalCorr`, `totalJuros`, `totalMulta` incrementados a cada período/dívida
- Resultado armazenado em `secoes[]` — array de `{ div, rows, saldoDiv }`
- Totais finais: `totalHonorarios`, `totalAtualizado`, `totalPago`, `saldoFinal` calculados após o loop

### Bloco de renderização PDF (linhas ~2159–2227 originais)

**Antes:** Resumo executivo horizontal com 3 células (Valor Original / Total Pago / Saldo Devedor), tabela única sobre `rows[]`.

**Depois:**
- **Resumo executivo vertical** (`resumoLinhas[]`): até 8 linhas — Valor Original, (+) Multa, (+) Honorários, (+) Correção, (+) Juros, (=) Total Atualizado, (-) Total Pago, (=) SALDO DEVEDOR FINAL. Linhas zeradas omitidas via spread condicional. Separador horizontal antes de "Total Atualizado". Linha final em bold/tamanho maior.
- **`renderTableHeader()`** — função interna com closure sobre `y` e `doc`, renderiza cabeçalho da tabela
- **`renderRows(rows)`** — função interna com closure, renderiza array de rows com zebra, overflow para nova página
- **Dívida única** (`dividasCalc.length === 1`): renderTableHeader + renderRows(secoes[0].rows) + bloco SALDO DEVEDOR ATUALIZADO (comportamento próximo ao anterior)
- **Múltiplas dívidas** (`dividasCalc.length > 1`): para cada seção — cabeçalho "DÍVIDA N: DESCRIÇÃO" (fundo #dcfce7), renderTableHeader, renderRows, bloco SUBTOTAL; ao final bloco TOTAL GERAL (fundo indigo)

## Variáveis-chave introduzidas

| Variável | Tipo | Descrição |
|----------|------|-----------|
| `totalCorr` | number | Soma de todas as correções monetárias absolutas (todos períodos, todas dívidas) |
| `totalJuros` | number | Soma de todos os juros calculados (todos períodos, todas dívidas) |
| `totalMulta` | number | Soma das multas do 1º período de cada dívida |
| `totalHonorarios` | number | honorariosPct% sobre totalAtualizado_sem_hon |
| `totalAtualizado` | number | PV + totalCorr + totalJuros + totalMulta + totalHonorarios |
| `saldoFinal` | number | totalAtualizado - totalPago |
| `secoes` | array | [{div, rows, saldoDiv}] — uma entrada por dívida elegível |
| `resumoLinhas` | array | Linhas do resumo executivo vertical (zeradas omitidas) |
| `pgtoRestantes` | array | Cópias dos pagamentos com campo .remaining para distribuição sequencial |

## Edge cases observados durante a implementação

1. **Pagamento no mesmo dia do início do período** (`periodoFim <= periodoInicio`): abate direto sem calcular correção/juros, usando `Math.min(pgto.remaining, saldo)` para não ultrapassar o saldo da dívida.
2. **Dívida já zerada** (`saldo <= 0`): loop interno interrompido com `break` — pagamentos restantes ficam com `.remaining` para a próxima dívida.
3. **Honorários sobre base correta**: calculados como `(pcSaldo + juros + multaVal) * honorariosPct / 100` no 1º período, mas o `totalHonorarios` do resumo executivo usa `totalAtualizado_sem_hon * honorariosPct / 100` para consistência global.
4. **Footer**: usa `indexador`, `jurosAM`, `multaPct` extraídos de `dividasCalc[0]` — mantendo a variável `divRef` para compatibilidade.
