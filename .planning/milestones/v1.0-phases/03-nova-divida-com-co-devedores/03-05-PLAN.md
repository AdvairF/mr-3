<objective>
Corrigir CR-04: após criar uma dívida via NovaDivida.jsx, a seção "Devedores desta Dívida" em
DetalheDivida exibe "Nenhum participante cadastrado" apesar do INSERT em devedores_dividas ter
sido bem-sucedido.

Root cause: listarParticipantes usa embedded join PostgREST (`devedor:devedores(...)`). Se o
PostgREST não encontrar o relacionamento FK no schema cache, retorna HTTP 400 — que sb() lança
como exceção. useDevedoresDividas captura essa exceção em silêncio (setError, mas nunca exibe),
mantendo participantes=[] e mostrando "Nenhum participante cadastrado."

Problema secundário: handleSalvar dispara toast.success independente de o vínculo ter sido
confirmado na tabela — sem guarda pós-loop.

Fixes:
1. Remover embedded join de listarParticipantes → query simples, sem dependência de FK no cache
2. Fazer join client-side em DevedoresDaDivida usando o prop devedores[] já carregado + expor error
3. Adicionar guarda em handleSalvar: após o loop, verificar que ao menos 1 row PRINCIPAL foi
   inserido em devedores_dividas — throw se não houver

Restrições absolutas:
- NÃO modificar adicionarParticipante
- NÃO modificar calcularSaldoDevedorAtualizado, calcularSaldosPorDivida, calcularTotalPagoPorDivida
- Commit local apenas, SEM push
</objective>
