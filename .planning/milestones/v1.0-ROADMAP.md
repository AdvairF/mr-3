## Overview

Separação do módulo Devedores em entidades Pessoas e Dívidas com relacionamento N:N real no banco, criação do Módulo Dívidas com tabela global e tela de detalhe, e adição da tela Nova Dívida com co-devedores e salvamento atômico. Elimina o JSONB embutido e estabelece a base para features futuras (pagamentos por dívida, contratos com parcelas).

## Milestone Summary

**Key Decisions:**

- `valor_total` column name (não `valor_original`) — matches devedorCalc.js field
- `art523_opcao TEXT` com CHECK constraint 3 valores — código usa nao_aplicar/so_multa/multa_honorarios
- `json_id_legado TEXT` bridge Date.now() IDs do JSONB para UUID rows na migration
- Aliases compat obrigatórios em objetos de dívida (indexador/juros_am/multa_pct/honorarios_pct) — adicionados por carregarTudo(), não pelo payload
- `devedor_id` em dividas = id do PRINCIPAL (desnormalizado) — manutenção manual em adições/remoções
- Motor de cálculo sequencial (Art. 354 CC) mantido sem migration de pagamentos_parciais
- Sem router — persistência de filtros via state React apenas
- Payload criarDivida usa APENAS nomes de colunas DB
- status: "em cobrança" explícito no payload (não depende de DEFAULT do banco)

**Issues Resolved:**

- CR-01: Aliases compat faltando em objetos dívida (indexador, juros_am, multa_pct, honorarios_pct)
- CR-02: Saldo individual por dívida (calcularSaldosPorDivida) corrigido
- CR-03: calcularTotalPagoPorDivida por dívida específica (não soma global)
- CR-04: listarParticipantes sem embedded join PostgREST — enriquecimento client-side

**Issues Deferred:**

- Limpeza Supabase: deletar pessoas "Criar %" e dívidas de teste (R$1/2/3/100)
- Config Vercel: desabilitar auto-deploy preview do submodule mr-3
- 36 quick tasks sem SUMMARY.md individual (acknowledged — work commitado)

**Technical Debt:**

- `devedores.dividas` JSONB ainda existe (não removido) — aguardando confirmação longa
- App.jsx ainda monolítico (~6.700 linhas) — extração progressiva continua nas próximas fases
