## 1. Problema

Durante UAT prod da Phase 7.8, usuário identificou gap no `DecomposicaoSaldoModal.jsx`: quando contrato tem pagamentos parciais registrados, o modal só mostra o saldo atualizado FINAL — não mostra QUANTO já foi deduzido do principal. Pra apresentar ao devedor (impressão A4), falta transparência do valor amortizado.

## 2. Escopo cirúrgico

**Fix único** — adicionar linha "Pagamento parcial" no array `linhas` do modal:

```javascript
if (totalPago > 0) {
  rows.push({
    label: "Pagamento parcial",
    value: -totalPago,       // valor negativo — fmtBRLSigned formata "- R$ X"
    signed: true,            // ativa fmtBRLSigned + valueColor (D-08 existente)
    bold: false,
  });
}
```

**Posição**: entre "Valor Original" (linha 85) e o bloco "Correção" (linha 87). Ordem financeira: subtrai pagamentos do principal antes de aplicar encargos visualmente.

**Propagação automática**: como `linhas` é consumido por 3 pontos (render React, `bodyRowsHtml` do print, `textoFormatado` do clipboard), **basta adicionar o push UMA VEZ** — os 3 consumidores pegam automaticamente via `linhas.map` / `linhas.forEach`.

**Extração do `totalPago`**: adicionar `const totalPago = Number(d.totalPago || 0);` junto das outras extrações no topo do componente (linhas ~45-58). Motor já expõe em `calcularDetalheEncargos` (linha 495 de `devedorCalc.js`) → adapter passa direto. Zero mudança no motor ou no adapter.

- **D-05** — **Posição no array**: DEPOIS de "Valor Original" (linha 85), ANTES do bloco `if (correcaoValor !== 0)` (linha 87). Representa ordem financeira: pagamento reduz o principal antes dos encargos serem aplicados visualmente.

- **D-06** — **Condicional de render (hide zero-value, precedente D-07 da 7.8)**: `if (totalPago > 0)` — linha SÓ aparece quando há pagamentos registrados. Consistente com o tratamento das outras linhas do modal.

- **D-07** — **Cor vermelha via helpers existentes** — `signed: true` dispara `fmtBRLSigned` + `valueColor` (retorna `#dc2626` pra valores negativos). Mesmo pattern usado pra correção negativa em D-08 da 7.8. Zero novo helper.

- **D-08** — **Impressão A4 (`bodyRowsHtml`)**: linha aparece automaticamente via `linhas.map`. Cor vermelha inline (`color: ${color}` condicional) já implementada no handler existente (`bodyRowsHtml`) — usa `color = l.signed && Number(l.value) < 0 ? "#dc2626" : "#000"`. Sem mudança no handler de impressão.

- **D-09** — **Clipboard (`textoFormatado`)**: linha aparece automaticamente via `linhas.forEach`. Prefix `"+ "` (não idx=0, não bold) resulta em `"+ Pagamento parcial: - R$ 300,00"` — label limpa + sinal no valor. **Sem ajuste do prefix logic** — a convenção do prefix representa "próxima linha da lista", não operador aritmético.

- **D-10** — **useMemo deps**: adicionar `totalPago` às dependências do `useMemo` de `linhas` (linha 119 atualmente) — senão não recalcula quando pagamentos mudam. Edge case: se o usuário registrar/excluir pagamento com modal aberto, valor deve atualizar.

- **D-11** — **Reconciliação aritmética** (matemática do modal): soma visual das linhas = saldoAtualizado sempre vale, porque `saldo = valor_original - totalPago + encargos` é invariante estrutural do motor Art.354 independente da ordem de aplicação. Nota de rodapé "Pode haver discrepância de R$ 0,01 por arredondamento" (D-11 da 7.8) cobre drift de float.

- **D-12** — **Edge cases validados**:
  - Contrato totalmente quitado (saldo=0, totalPago=valorOriginal): linha aparece, componentes zerados escondidos, Total=R$ 0,00 aparece. Matemática: `V - V + 0 = 0` ✓.
  - Pagamento = valor original + encargos acumulados: linha aparece, encargos aparecem, Total=saldo remanescente. Semanticamente claro.
  - Zero pagamentos: linha NÃO aparece (consistente com D-06).
  - Pagamento > valor original: impossível pelo motor (`if (saldo <= 0) break` trunca amortização). Non-issue.

### `totalPago` no adapter output — CONFIRMADO

`calcularDetalheEncargos:495` em `devedorCalc.js`:
```javascript
const totalPago = (pagamentos || []).reduce((s, p) => s + (parseFloat(p.valor) || 0), 0);
```
Retornado no shape (linha 516). Adapter `calcularDetalheEncargosContrato(dividasDoContrato, pagamentos_divida, hoje)` passa `pagamentos_divida` direto → `totalPago` = soma de `pagamentos_divida.valor`. Match semântico com a expectativa do usuário.

### Helpers existentes reutilizados

- `fmtBRLSigned(v)` — já formata negativos como `"- R$ X"` (linha 15-23 do modal)
- `valueColor(v)` — já retorna `#dc2626` pra negativos (linha 25-27)
- Inline color no `bodyRowsHtml` (linha 160) — já usa `const color = l.signed && Number(l.value) < 0 ? "#dc2626" : "#000";`

Zero novo helper. Zero novo CSS. Tudo reaproveitado do D-08 da 7.8 (precedente).
