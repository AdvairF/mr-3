## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Elevation of Privilege (T-INSERT) | handleSalvar criarDivida | mitigate | RLS Supabase applies to dividas table; Bearer token via `sb()` in supabase.js line 17; user must be authenticated to reach this view |
| T-03-03-02 | Tampering (T-UUID) | adicionarParticipante dividaId | mitigate | `dividaId: String(novaDiv.id)` — UUID comes from Supabase response, validated as truthy before loop; not from user input or URL param; no encodeURIComponent needed for POST body values |
| T-03-03-03 | Tampering | devedor_id desnormalizado | accept | devedor_id is taken from `pessoas.find(p => p.papel === "PRINCIPAL" && p.devedor_id != null)?.devedor_id` — a BIGINT from the loaded allDevedores state (already validated by Supabase when loaded); not from raw user text input |
| T-03-03-04 | Information Disclosure | Toast on save error | accept | `toast.error("Erro ao salvar: " + e.message)` — Supabase error messages may expose table names; risk accepted (internal tool, authenticated users only) |
