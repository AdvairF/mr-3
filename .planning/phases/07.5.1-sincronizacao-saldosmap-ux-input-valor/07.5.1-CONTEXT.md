## Phase Boundary

**Corrige 3 bugs descobertos no UAT da Phase 7.5, todos cirúrgicos.** O motor Art.354 (`devedorCalc.js` + `pagamentos.js`) **NÃO é tocado** — invariante estrutural preservada, os 9 testes de regressão TJGO continuam válidos sem teste novo.

**Bugs no escopo:**

### Bug A — R$ 39,90 fantasma
Quando usuário reagenda parcela para data futura via "Editar parcelas" (feature da Phase 7.5), a coluna Saldo mostra penalidade fantasma (~R$ 39,90) mesmo com a parcela já no futuro. Saldo puro deveria ser exibido.

### Bug B — Saldo não recalcula após edição de valor
Quando usuário edita valor de parcela (ex: R$ 300 → R$ 500), coluna Valor atualiza imediatamente mas coluna Saldo mantém o valor antigo (R$ 300) até recarregar a página (F5).

### Bug C — Input valor não apaga zero
No componente `TabelaParcelasEditaveis.jsx` (criado na Phase 7.5), ao apagar todo o conteúdo do campo valor com backspace, o campo mostra "0" e qualquer keystroke seguinte concatena ao zero ("050" em vez de "50").

**Fora do escopo:**
- Otimização via hash signature no useEffect (deferido pra hipotética Phase 7.5.2 se UAT detectar lentidão em contratos com 50+ parcelas)
- Refactor de `saldosMap` para usar Map/WeakMap
- Qualquer mudança no motor Art.354
- Evento `parcelas_editadas` em `contratos_historico` (ainda deferido à futura rastreabilidade forense)
- Qualquer alteração de schema em `dividas` ou `pagamentos_divida`
- Novos testes para o motor (9 atuais são suficientes — motor não é tocado)

<decisions>
## Implementation Decisions

### Root Cause — Bugs A + B (MESMA raiz, 1 linha)

- **D-01 (travado):** `useEffect` em `src/mr-3/mr-cobrancas/src/components/DetalheContrato.jsx:173` (linha que calcula `saldosMap`) tem deps `[expandedDoc, hoje]` — falta `dividas`. Quando `atualizarParcelasCustom` (Phase 7.5) termina, `onCarregarTudo` atualiza `allDividas` no `App.jsx`, a prop `dividas` chega nova no `DetalheContrato`, mas o useEffect não dispara porque as deps atuais não mudaram. Pior: o update usa `setSaldosMap(prev => ({...prev, ...new}))` que faz merge — IDs existentes nunca são sobrescritos até um novo expand. **Fix: adicionar `dividas` às deps.**
- **D-02 (travado):** Bug A é o mesmo bug. Saldo calculado quando a parcela estava vencida fica cached; reagendar pra futuro deveria zerar penalidade, mas o componente nunca re-chama o motor pra aquele ID.
- **D-03 (travado):** Motor Art.354 **está correto**. Em `devedorCalc.js:238`, o bloco de penalidades só executa se `periodoInicio < hoje`. Parcela futura → motor retorna saldo puro. Verificado linha-por-linha. Bug é 100% no componente.

### Deps do useEffect — abordagem escolhida

- **D-04 (travado):** Deps simples `[expandedDoc, hoje, dividas]`. **NÃO** implementar hash signature otimizado nesta phase.
  - **Por quê:** fix real, overhead aceito para esta sessão.
  - **Trade-off aceito:** re-cálculo completo do saldosMap para o `expandedDoc` atual toda vez que `dividas` muda. Em contratos com 50+ parcelas, pode haver latência visível. Se UAT detectar, Phase 7.5.2 introduz hash signature.

### Bug C — Input não apaga zero

- **D-05 (travado):** Root cause em `src/mr-3/mr-cobrancas/src/components/TabelaParcelasEditaveis.jsx`. Value prop e state estão como Number puro. Quando usuário apaga tudo: `Number("") = 0` → state `0` → input mostra `"0"`. Browser concatena keystrokes ao "0" preexistente.
- **D-06 (travado):** Fix correto — **state local string** em vez de Number puro. Permite campo vazio durante edição. NÃO é workaround visual.
  - **Discretion do planner:** escolher entre (a) state auxiliar `valoresStr: string[]` paralelo ao `parcelas`, ou (b) `value={p.valor_total === 0 && editandoVazio ? "" : String(p.valor_total)}`. O importante é permitir campo vazio durante edição sem quebrar a prop `onChange` downstream.

### Motor Art.354 — invariante estrutural

- **D-07 (travado):** `src/mr-3/mr-cobrancas/src/utils/devedorCalc.js` e `src/mr-3/mr-cobrancas/src/services/pagamentos.js` **NÃO sofrem diff**. Verificável com `git diff HEAD~N -- devedorCalc.js pagamentos.js` vazio.
- **D-08 (travado):** 9 testes regressão TJGO (`npm run test:regressao`) continuam passando sem teste novo — invariante estrutural substitui teste unitário adicional.
