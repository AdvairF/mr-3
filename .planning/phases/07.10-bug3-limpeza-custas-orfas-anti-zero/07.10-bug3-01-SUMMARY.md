## Bug original
Phase 7.9 deixou mecânica Q2 conservative em `excluirCusta` (services/contratos.js:646-658): quando o operador deleta a última custa de uma dívida-fantasma `_so_custas:true`, o array `custas` vira `[]` mas a dívida-fantasma persiste com `valor_total=0`, virando órfã visual no Módulo Dívidas (linha R$ 0,00 / R$ 0,00).

Investigação SQL ampliada (queries 1-14, TASK 0 operador 2026-04-30) mapeou:
- **7 órfãs MENDES** devedor_id=25 contrato `ce7b8d47` — `_so_custas=true` + `jsonb_array_length(custas)=0` (mecânica Q2 conservative confirmada)
- **3 dívidas advair** id=8 — assumidas inicialmente como "regulares legacy lixo" (esperado em `regulares_sem_contrato` por contrato_id NULL)

## Solução — 3 partes

### Parte 1: Cleanup imediato (Sub-cleanup A)
SQL DELETE pattern genérico em prod via Supabase SQL Editor:
```sql
DELETE FROM dividas
WHERE _so_custas = true
  AND jsonb_array_length(COALESCE(custas, '[]'::jsonb)) = 0
  AND devedor_id = 25;
-- DELETE 7
```

Defesa em profundidade: ABORT GATE pre-cleanup (Query 7+14+AJUSTE 1) + BACKUP SNAPSHOT pré-DELETE (`dividas_backup_pre_710bug3`, dropado pós-UAT PASS).

### Parte 2: Bugfix `excluirCusta` cascade DELETE
Substituiu Q2 conservative por cascade DELETE quando array vira vazio E `_so_custas=true`. Junction `devedores_dividas` CASCADE (Migration 002:161 preservada cumulative) propaga limpeza automaticamente.

### Parte 3: Anti-zero validation (server+client)
- **Server-side** (`criarCusta` em `services/contratos.js`): throw `"Valor da custa deve ser maior que zero"` se `valor <= 0`
- **Client-side** (handler form em `DetalheContrato.jsx`): bloqueia submit com mesma string

## Fix-forward dbDelete signature mid-execução

UAT SC3 prod 2026-04-30 detectou bug crítico no Task 2: `dbDelete("dividas", \`id=eq.${...}\`)` passou string com prefix `id=eq.` como 2º arg. PostgREST 400: `invalid input syntax for type uuid: 'id=eq.<UUID>'`.

**Causa raiz:** `dbDelete` em `supabase.js:76` constrói o filtro internamente (`(t, id) => sb(t, "DELETE", null, "?id=eq.${id}")`). 2º arg deve ser **UUID puro**, espelhando L393 `excluirContrato` + L471 `excluirDocumento` (preexistentes, prod OK cumulative).

**Fix:** 1 linha mudada em `services/contratos.js:671` — `dbDelete("dividas", dividaId)`. Commit submódulo `80f8f1c` + chore bump pai `31116ba` (TAG TARGET).

Lição: helpers com prefix interno (dbDelete, dbUpdate) exigem audit pré-execute dos callsites preexistentes pra padronizar — não basta inferir do helper vizinho (`dbGet` tem semântica diferente: aceita query string completa).
