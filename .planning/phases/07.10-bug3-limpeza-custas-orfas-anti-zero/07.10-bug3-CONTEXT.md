## 1. Bug summary

### Sintoma observado
Módulo Dívidas filtro "mend" exibe **20 linhas** para devedor MENDES E MENDES LTDA (CNPJ 11.723.033/0001-86, devedor_id=25), enquanto contrato ce7b8d47 mostra apenas **13 custas reais** (soma R$ 2.821,33 ≈ R$ 2.821,81 com arredondamento). Diferença = 7 linhas R$ 0,00 / R$ 0,00 órfãs.

Investigação adicional revelou 3 órfãs de teste em devedor "advair" id=8 (lixo de UAT exploratório).

### Mecânica raiz confirmada
Phase 7.9 D-23 estabeleceu modelo "1 custa avulsa = 1 dívida-fantasma" (`_so_custas:true`, `custas:[1 item]`).

Phase 7.9 Q2 conservative em `excluirCusta` (`services/contratos.js:646-658`):
```js
/**
 * Exclui uma custa do array JSONB. Se ficar vazio E divida._so_custas === true,
 * mantém a dívida vazia (Q2 conservative — evita cascade de effects não auditados).

  // ... lê divida

  await atualizarDivida(dividaId, { custas: novasCustas });
  // Q2 — dívida avulsa vazia é preservada conservativamente (comment locked).

```

## 2. Solução — 3 partes

### Parte 1: Cleanup imediato (SQL ops)

#### Sub-cleanup A — 7 órfãs MENDES (via critério genérico do mecanismo)
```sql
DELETE FROM dividas
WHERE _so_custas = true
  AND jsonb_array_length(COALESCE(custas, '[]'::jsonb)) = 0
  AND devedor_id = 25;
```
**Critério:** dívidas-fantasma com array vazio do devedor MENDES E MENDES (id=25). Captura o pattern do bug Q2 conservative confirmado pela investigação.

**Pre-confirmação:** rodar Query 14 + Query 7 imediatamente antes do DELETE pra confirmar que count ainda é 7 (não houve mudança desde 2026-04-30).

#### Sub-cleanup B — 3 órfãs advair (via lista explícita de IDs)
```sql
DELETE FROM dividas
WHERE id IN (
  '85066e3d-289d-4cd2-9b00-09e8f219a5cd',
  'd82841e3-33d3-4c36-8a8d-93c7398530a1',
  '6cd6e321-62cd-4f68-b3aa-65d0f7a4f00f'

```

**Justificativa caso a caso (não regra genérica):**
- Devedor "advair" id=8 (próprio operador)
- Sem credor_id, sem documento_id, sem tipo_titulo (campos NULL)
- Valor R$ 150,00 fixo nos 3
- Criadas no mesmo dia (2026-04-29) — UAT exploratório
- Operador confirmou: lixo de teste sem valor jurídico

**NÃO usar critério genérico** "DELETE dividas WHERE contrato_id IS NULL" — outros NULLs podem ser legacy pré-Phase 5 com valor histórico. DELETE explícito por ID evita false positive.

### Parte 2: Bugfix `excluirCusta` (cascade DELETE)

Substitui Q2 conservative Phase 7.9 por cascade explícito.

**Localização:** `src/services/contratos.js:650-658`

**Mudança:**
```js
// ANTES
export async function excluirCusta(contratoId, dividaId, custaId) {
  const dividaRows = await dbGet("dividas", `id=eq.${encodeURIComponent(dividaId)}&limit=1`);
  const divida = Array.isArray(dividaRows) ? dividaRows[0] : null;
  if (!divida) throw new Error("divida não encontrada");
  const custasAtuais = Array.isArray(divida.custas) ? divida.custas : [];
  const novasCustas = custasAtuais.filter(c => String(c.id) !== String(custaId));
  await atualizarDivida(dividaId, { custas: novasCustas });
  // Q2 — dívida avulsa vazia é preservada conservativamente (comment locked).
}

// DEPOIS
export async function excluirCusta(contratoId, dividaId, custaId) {
  const dividaRows = await dbGet("dividas", `id=eq.${encodeURIComponent(dividaId)}&limit=1`);
  const divida = Array.isArray(dividaRows) ? dividaRows[0] : null;
  if (!divida) throw new Error("divida não encontrada");
  const custasAtuais = Array.isArray(divida.custas) ? divida.custas : [];
  const novasCustas = custasAtuais.filter(c => String(c.id) !== String(custaId));
  // Phase 7.10.bug3 D-pre-2 — substitui Q2 conservative Phase 7.9.
  // Quando array vira vazio E divida._so_custas === true, deletar dívida-fantasma
  // (cascade automático via junction devedores_dividas FK CASCADE — Migration 002:161).
  if (novasCustas.length === 0 && divida._so_custas === true) {
    await dbDelete("dividas", `id=eq.${encodeURIComponent(dividaId)}`);
  } else {
    await atualizarDivida(dividaId, { custas: novasCustas });
  }
}
```

**Side-effects auditados:**
- Junction `devedores_dividas` tem `divida_id UUID NOT NULL REFERENCES dividas(id) ON DELETE CASCADE` (Migration 002 L161) → DELETE da dívida-fantasma propaga pra junction. Seguro.
- Tabela `pagamentos_divida` — dívida-fantasma `_so_custas:true` não tem pagamentos parciais (custas integral-only D-22). DELETE seguro.
- Eventos andamento — vinculados a contrato_id, não divida_id. Sem impacto.
