## Implementation Decisions

### Bug A — Renderização do Histórico de Pagamentos (linhas ~805-881)

- **D-01:** Declarar `const isPagamento = ['pagamento_recebido', 'pagamento_revertido'].includes(evento.tipo_evento)` logo após `isCriacao` (linha 806), antes do cálculo de `diffEntries`.

- **D-02:** Modificar o cálculo de `diffEntries` (linha 810) para incluir `&& !isPagamento` na guarda ternária:
  ```js
  const diffEntries = !isCriacao && !isPagamento
    ? Object.entries(snap).map(([campo, val]) => ({
        campo,
        antes:  String(val?.antes  ?? ""),
        depois: String(val?.depois ?? ""),
      }))
    : [];
  ```
  Motivo: evitar computação desnecessária com "— → —" para eventos de pagamento.

- **D-03:** Adicionar ramo JSX específico para `isPagamento` logo após o bloco `{isCriacao && ...}` (linha 844) e antes do bloco `{!isCriacao && diffEntries.length > 0 && ...}` (linha 862):
  ```jsx
  {isPagamento && (
    <div style={{ fontSize: 13, color: "#374151", lineHeight: 1.7 }}>
      <div>Valor: {fmtBRL(snap.valor)}</div>
      <div>Data: {fmtData(snap.data_pagamento)}</div>
      <div>
        Parcelas: {buildParcelasText(snap.parcelas_ids, dividas)}
        {" "}({snap.parcelas_amortizadas ?? snap.parcelas_ids?.length ?? 0} amortizada(s))
      </div>
    </div>
  )}
  ```
  Notas:
  - `snap.data_pagamento` é coluna DATE → usar `fmtData` (correto, não fmtDataHora)
  - `snap.parcelas_amortizadas` existe só em `pagamento_recebido`, não em `pagamento_revertido` → fallback `?? snap.parcelas_ids?.length ?? 0`
  - `buildParcelasText(parcelas_ids, dividas)` já existe no componente (Phase 7, linha ~385) — reutilizar para mostrar "Parc. 1/6, 2/6" como no resto da UI

- **D-04:** O guard existente `{!isCriacao && diffEntries.length > 0 && ...}` (linha 862) NÃO precisa de alteração adicional — com D-02, `diffEntries` já será `[]` para eventos de pagamento, então `diffEntries.length > 0` será false.

### Bug B — Timezone em created_at do Histórico (linha 840)

- **D-05:** Adicionar função `fmtDataHora(iso)` **imediatamente após** `fmtData` (linha 13):
  ```js
  function fmtDataHora(iso) {
    if (!iso) return "—";
    try {
      const d = new Date(iso);
      if (isNaN(d.getTime())) return "—";
      return d.toLocaleDateString("pt-BR", { timeZone: "America/Sao_Paulo" });
    } catch { return "—"; }
  }
  ```
  Motivo: `new Date(null)` retorna epoch sem jogar exceção — validar `isNaN(d.getTime())` além do try/catch.

- **D-06:** Substituir `fmtData(evento.created_at)` por `fmtDataHora(evento.created_at)` na **única** chamada da linha 840.

- **D-07:** NÃO alterar `fmtData` nem nenhuma das outras chamadas existentes — todas usam colunas DATE (data_pagamento, data_vencimento, data_emissao) que funcionam corretamente com o slice.
