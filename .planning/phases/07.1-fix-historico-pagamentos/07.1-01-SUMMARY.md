## Changes Made

### Bug B — fmtDataHora (timezone fix)

**Line 14 — New function inserted immediately after fmtData:**

```js
function fmtDataHora(iso) {
  if (!iso) return "—";
  try {
    const d = new Date(iso);
    if (isNaN(d.getTime())) return "—";
    return d.toLocaleDateString("pt-BR", { timeZone: "America/Sao_Paulo" });
  } catch { return "—"; }
}
```

**Line 849 — Replaced single call site:**
- Before: `{fmtData(evento.created_at)}`
- After: `{fmtDataHora(evento.created_at)}`

`fmtData` and all its other call sites (data_pagamento, data_vencimento, data_emissao) remain unchanged — those columns are DATE type which works correctly with slice.
