## Lessons learned

1. **Strategy A (shared wrapper delegation) > Strategy B (direct edits) para paste-friendly varredura ampla** — atualizar shared `<Inp>` wrapper internalmente cobre callsites consumers automaticamente sem touchar component files. Pattern DRY que previne migration drift e protege D-pre-7 cumulative invariant (DetalheContrato.jsx intocado). Reutilizável para outras varreduras amplas que afetam shared wrappers.

2. **Encargos motor parity via STRING preservation** — `<InputBR type="value">` mantém value como STRING no state, não converte para number. Crítico porque `parseFloat()` consumers em `devedorCalc.js` esperam string. Lição transferível: ao adicionar wrappers em fluxos com motor de cálculo downstream, sempre preservar tipo do dado original (string/number/etc).

3. **prebuild hook como gate automático** — `npm run prebuild` rodar `test:regressao` antes de build dá camada extra de proteção. Falha em build se motor regression. Pattern reutilizável para outras phases com motor crítico.

4. **Plan accuracy fix-forward via Strategy A** — plan original listou DetalheContrato.jsx + NovaCustaModal.jsx como direct callsites, mas auditoria do executor revelou eram `<Inp>` callsites cobertos automaticamente via Strategy A. Não é desvio — é refinamento DRY. Lição: planners devem auditar shared wrappers (`<Inp>`, etc) ANTES de listar direct callsites em components que possam usá-los.

5. **20 parser tests cobrem casos edge BR** — formatos como "R$ 1.234,56", "1234,56", "1234.56", strings vazias, null, NaN, Infinity. Test count gerou alta confidence pre-merge. Vitest framework confirmado via package.json (4.1.4).
