## Lessons learned

1. **Briefing original confundia UPDATE com criação NOVA** — D-pre-7 inicial mencionava `DetalheContrato.handleSalvar` mas esse handler é EDIÇÃO (chama `editarContrato`). Investigação grep pré-discuss revelou que criação NOVA está em `NovoContrato.handleSalvar:44` + `services/contratos.js:111 criarContrato`. Lição transferível: NUNCA acreditar em path/linha do briefing sem verificar grep — handlers com mesmo nome em components diferentes são fonte comum de confusão.

2. **fire-and-forget em service layer cria assertion gotcha** — `criarContrato` chama `registrarEvento` (fire-and-forget) gerando 2º call em `dbInsert`. Tests com `toHaveBeenCalledTimes(1)` quebram. Solução: assertion específica do PRIMEIRO call com `mock.calls[0]`. Lição transferível: services com side effects fire-and-forget precisam assertions específicas, não count-based.

3. **Defense in depth UI + service** — D-pre-10 enforced UI valida ANTES do submit (UX) + service-layer guarda ANTES do INSERT (integridade). Ambos throwam mensagem semântica. Pattern previne futuros consumers (scripts, tests, novos componentes) de bypass. Reutilizável em criação de qualquer entidade com FKs obrigatórios.
