## Lessons learned

1. **Soft delete BYPASSA todas as FKs** — não há DELETE físico, então nenhuma FK constraint é violada. Esta é a beleza da decisão D-pre-1: contorna o bug raiz sem precisar fixar 12 FKs uma a uma. Pattern aplicável a outros domínios com FK chains complexas.

2. **Comentários audit trail vs strict-mode shields** — quando o comentário PRECISA mencionar literal o token que está sendo evitado (D-pre-11 audit trail), o shield strict-mode dá false positive. Solução: usar shield regex anchored ao callsite real (JOIN string), não substring genérico. Lição transferível para futuras phases com exceções intencionais.

3. **Bugs pré-existentes em features dependentes bloqueiam UAT visual** — Phase 7.10.bug typo `[object Object]` em ProcessosJudiciais cadastro impediu validação visual SC5/SC22. Phases que migram fonte de dados em features dependentes devem audit pré-execute se feature alvo é utilizável em prod (lição transferível, registrada em feedback `proc_judiciais_create_bug_preexisting`).
