## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — pure inline React styles with hex values |
| Icon library | Unicode/emoji inline (◆ ✏️ ▲ ▼ ←) — no icon library dependency |
| Font | 'Space Grotesk' (headings, amounts) · 'Plus Jakarta Sans' (body, labels, inputs) |

No shadcn. No Tailwind. No CSS modules. All styling via `style={{ }}` objects with explicit hex values.

---

## Spacing Scale

All values are multiples of 4. Source: existing patterns in DetalheContrato.jsx and DiretrizesContrato.jsx.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (2px 8px vertical/horizontal), inline label margin |
| sm | 8px | Input bottom margin, label-to-input gap (marginBottom: 4 → 8), compact row gaps |
| md | 16px | Card internal padding (horizontal), section label bottom margin |
| lg | 24px | Card padding (24px uniform — matches header card), section vertical gap |
| xl | 32px | Page bottom padding, major section separation |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Timeline dot column: fixed 32px wide (left gutter for timeline icon + line)
- Timeline vertical line: 2px wide, positioned at `left: 16px` from left of gutter (centers the 2px line within 32px gutter — multiple of 4)
- Touch-safe buttons: minimum height 36px for Salvar/Cancelar in edit mode
- Inp component internal padding: `8px 10px` — frozen legacy API, do not apply spacing rules
- DiretrizesContrato grid gap: 12px (matches existing component — do not change)

### Surface 2 — Header Card in Edit Mode

Activated when `editando === true`. The entire header card content changes. The card container border switches from `1px solid #e8f0f7` to `1px solid #c7d2fe` and background stays `#fff`.

Layout in edit mode:
```
[label: REFERÊNCIA]
[text input: referencia — full width — Inp component]

[label: CREDOR]                    [label: DEVEDOR]
[select: credor_id — 1fr]          [select: devedor_id — 1fr]

[DiretrizesContrato component — full width — reused as-is]

[Cancelar btn]  [Salvar btn]
```

Grid layout: 2-column grid (1fr 1fr) for credor/devedor selects. Referência input spans full width (gridColumn: "1/-1"). DiretrizesContrato spans full width below the grid.

Field specs:
- Referência: `<Inp label="Referência" value={referencia} onChange={setReferencia} span />` — Inp component from ui/Inp.jsx
- Credor: `<Inp label="Credor" value={credorId} onChange={setCredorId} options={credoresOptions} />` — options array of {v: id, l: nome}
- Devedor: `<Inp label="Devedor" value={devedorId} onChange={setDevedorId} options={devedoresOptions} />` — options array of {v: id, l: nome}
- DiretrizesContrato: `<DiretrizesContrato value={encargos} onChange={handleEncargos} />` — zero changes to that component

Button row style:
```
style={{ display: "flex", gap: 8, justifyContent: "flex-end", marginTop: 16 }}
```

Buttons:
- Cancelar: `<Btn outline color="#64748b" sm onClick={handleCancelar}>Cancelar</Btn>`
- Salvar: `<Btn color="#4f46e5" sm disabled={salvando} onClick={handleSalvar}>{salvando ? <Spinner /> : "Salvar"}</Btn>`

```jsx

  return (
    <span style={{
      display: "inline-block",
      width: 12,
      height: 12,
      border: "2px solid rgba(255,255,255,.3)",
      borderTop: "2px solid #fff",
      borderRadius: "50%",

// Requires: @keyframes spin { to { transform: rotate(360deg) } } in global CSS or inline style tag
```

Note: `salvando` state disables the Salvar button (opacity .6, cursor not-allowed — handled by Btn disabled prop).

## Interaction Spec

### INT-01 — Edit Mode Activation

1. User clicks "Editar Contrato" button in header card read mode.
2. `editando` state toggles to `true`.
3. Header card content replaces immediately (no animation) with edit form.
4. Card border changes from `#e8f0f7` to `#c7d2fe`.
5. Form fields pre-populated with current contrato values.
6. DiretrizesContrato renders below credor/devedor with current encargos values.
7. Salvar and Cancelar buttons appear in bottom-right of header card.

### INT-02 — Cancel Edit

1. User clicks "Cancelar".
2. `editando` toggles to `false` immediately.
3. Header card returns to read mode with original values (no changes persisted).
4. Card border returns to `#e8f0f7`.
5. No toast.

### INT-03 — Save Edit (no cascade — only referencia or encargos changed)

1. User clicks "Salvar".
2. `salvando` set to `true` → Salvar button shows spinner, disabled.
3. `editarContrato(contrato.id, payload)` called.
4. On success: `registrarEvento('edicao', snapshot_campos_alterados)` called.
5. `toast.success("Contrato atualizado.")` fires.
6. `editando` set to `false`, header returns to read mode with updated values.
7. `salvando` set to `false`.
8. On error: `toast.error("Erro ao salvar contrato: " + e.message)`, `salvando` set to `false`, edit mode stays open.

### INT-04 — Save Edit (with cascade — credor_id or devedor_id changed)

1. User clicks "Salvar".
2. System detects credor_id or devedor_id changed vs original.
3. BEFORE calling any service: `window.confirm()` shown with exact copy:
   - Devedor changed: `"Alterar o devedor vai atualizar N parcelas (incluindo quitadas). Confirmar?"`
   - Credor changed: `"Alterar o credor vai atualizar N parcelas (incluindo quitadas). Confirmar?"`
   - Both changed: two sequential confirms (devedor first, then credor) — OR single confirm listing both fields: `"Alterar credor e devedor vai atualizar N parcelas (incluindo quitadas). Confirmar?"`
   - N = count of all dividas where documento_id IN (documentos do contrato)
4. If user cancels confirm: no save, no cascade, edit mode stays open.
5. If user confirms: `salvando` set to `true`, Salvar button shows spinner.
6. `cascatearCredorDevedor(contrato.id, {credor_id, devedor_id})` called (updates documentos_contrato + dividas including saldo_quitado=true).
7. `editarContrato(contrato.id, payload)` called for header fields.
8. `registrarEvento('edicao', snapshot_campos_alterados)` called.
9. `toast.success("Contrato e parcelas atualizados.")` fires.
10. `editando` set to `false`, header returns to read mode.
11. `onCarregarTudo()` called to refresh parent App state (allDividas, allContratos).
12. On any error: `toast.error("Erro ao propagar alteração: " + e.message)`, `salvando` false, edit mode stays open.

### INT-05 — History Section Toggle

1. "Histórico ▼" toggle row is always visible in DetalheContrato (after documentos section).
2. Default state: `historicoAberto === false` (collapsed).
3. First click: `historicoAberto` sets to `true`.
4. Lazy load fires: `listarHistorico(contrato.id)` called.
5. During load: "Carregando histórico..." shown inside expanded container.
6. On load complete with rows: timeline renders (newest event first — ORDER BY created_at DESC).
7. On load complete with zero rows: empty state renders ("Sem histórico disponível").
8. On error: `toast.error("Erro ao carregar histórico: " + e.message)`.
9. Subsequent toggles: do NOT re-fetch — use cached `historico` state. Re-fetch only on `contrato.id` change.
10. Second click: collapses, toggle shows "▼" again.

### INT-06 — Cascade Count Calculation

N parcelas count for confirm message is calculated client-side before showing confirm:
```js
const parcelas = dividas.filter(d =>
  documentos.some(doc => String(doc.id) === String(d.documento_id))
);
const N = parcelas.length;
```
`dividas` prop already available in DetalheContrato. `documentos` already in local state.

## State Machine

New state variables added to DetalheContrato:

| State Variable | Type | Initial | Description |
|----------------|------|---------|-------------|
| `editando` | boolean | false | Header card in edit mode |
| `salvando` | boolean | false | Save in progress (disables Salvar) |
| `editForm` | object | {} | Controlled form state: {referencia, credor_id, devedor_id, encargos} |
| `historicoAberto` | boolean | false | History section expanded |
| `historico` | array | [] | Cached events from contratos_historico |
| `historicoLoading` | boolean | false | First-load flag for history |

State machine transitions:

```
read mode (editando=false)
  → [click "Editar Contrato"] → edit mode (editando=true)
    → [click "Cancelar"] → read mode (no changes)
    → [click "Salvar"] → saving (salvando=true)
      → [error] → edit mode (salvando=false, toast.error)
      → [success] → read mode (salvando=false, toast.success, editando=false)

historico collapsed (historicoAberto=false)
  → [click toggle] → expanded (historicoAberto=true)
    → lazy load fires once
    → [click toggle] → collapsed (no re-fetch on reopen)
```
