### Novas invariantes 7.10a

- **D-26** — **ProcessosJudiciais lê APENAS de pagamentos_divida pós-7.10a**. Callsite App.jsx L8478 fixa `pagamentos={allPagamentosDivida}`. Shield 22 (positivo + negativo) blinda contra refactor futuro acidentalmente trazer `allPagamentos` legacy de volta.

- **D-27** — **Filter por divida_id (não devedor_id direto)**. Pattern: lookup divida_id ∈ dividas-do-devedor, então filter. Documentado inline em comment. Pattern reutilizável para futuras phases 7.10b/c/d se houver consumer com lógica de filtro por devedor.

- **D-28** — **Drift histórico aceito por design**. Pagamentos via UI legacy "Registrar pagamento parcial" (App.jsx L2537/2558/2574) ainda escrevem em `pagamentos_parciais`. Trade-off aceito: cortar writes é phase própria, misturar com migração de leitura quebra atomicidade. Documentar visivelmente em SUMMARY + flag pra usuário esperar drift até unificação.

- **D-29** — **Motor compatível verificado**. `calcularSaldoDevedorAtualizado` (devedorCalc L59) lê apenas `p.data_pagamento + p.valor`. Não consulta `devedor_id` nem `divida_id`. Shape `pagamentos_divida` (que tem ambos data_pagamento+valor) é compatível. Motor distribui pagamentos FIFO pelas dividas do devedor — mesmo comportamento de antes.

### Shape pagamentos_divida (locked, schema atual)

```sql
CREATE TABLE pagamentos_divida (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  divida_id       UUID NOT NULL REFERENCES dividas(id) ON DELETE CASCADE,
  data_pagamento  DATE NOT NULL,
  valor           NUMERIC(15,2) NOT NULL,
  observacao      TEXT
)
```

Diferenças vs `pagamentos_parciais` (legacy):
- `pagamentos_parciais.devedor_id` (FK direto) → `pagamentos_divida.divida_id` (FK indireto via divida)
- Schema mais normalizado: pagamento conhece a dívida específica que está amortizando

### Shield 23 — fixture esperada (planner detalha)

```js
// Fixture: devedor com 2 dívidas + 3 pagamentos (1 em cada dívida + 1 spread)
const devedor = { id: "dev-1", dividas: [{ id: "div-A" }, { id: "div-B" }] };
const pagamentos = [
  { id: "p1", divida_id: "div-A", data_pagamento: "2026-01-01", valor: 100 },
  { id: "p2", divida_id: "div-B", data_pagamento: "2026-02-01", valor: 200 },
  { id: "p3", divida_id: "div-A", data_pagamento: "2026-03-01", valor: 50 },
];
// Filter via Set lookup (pattern novo)
const dividaIdsDoDevedor = new Set(devedor.dividas.map(d => String(d.id)));
const novo = pagamentos.filter(p => dividaIdsDoDevedor.has(String(p.divida_id)));
// Filter equivalente legacy (simulando shape antigo com devedor_id sintético)
const sintetico = pagamentos.map(p => ({ ...p, devedor_id: devedor.id }));
const legacy = sintetico.filter(p => p.devedor_id === devedor.id);
// Asserção: equivalência exata
expect(novo).toEqual(legacy);
expect(novo).toHaveLength(3);
```
