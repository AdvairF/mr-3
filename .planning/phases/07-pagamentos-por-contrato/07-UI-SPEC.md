## Component Specifications

### 3b — "Registrar Pagamento" Button + Inline Form

**Trigger button (visible when `registrandoPagamento === false`):**
- Component: `<Btn color="#0d9488" sm onClick={() => setRegistrandoPagamento(true)}>Registrar Pagamento</Btn>`
- Placement: `<div style={{ marginTop: 0, marginBottom: 16 }}>` immediately after financial summary card
- No icon prefix — plain text label only

**Inline form card (visible when `registrandoPagamento === true`):**
- Container: `<div style={{ background: "#fff", borderRadius: 16, padding: "16px 24px", marginBottom: 16, border: "1px solid #c7d2fe" }}>`
- Matches edit-mode card border exactly (#c7d2fe)
- Field layout: 2-column grid for Data + Valor, full-width for Observação

**Field order (Data first per CONTEXT.md Claude's Discretion):**
1. Row 1 (grid 1fr 1fr, gap 12):
   - Data: `<Inp label="Data" type="date" />` — max value = hoje (HTML max attribute)
   - Valor (R$): `<Inp label="Valor (R$)" type="number" step="0.01" min="0" />`
2. Row 2 (full width):
   - Observação: `<Inp label="Observação" />` — optional field

**Actions row:**
- `<div style={{ display: "flex", gap: 8, justifyContent: "flex-end", marginTop: 16 }}>`
- Cancelar: `<Btn outline color="#64748b" sm onClick={() => setRegistrandoPagamento(false)} disabled={salvandoPagamento}>Cancelar</Btn>`
- Salvar: `<Btn color="#0d9488" sm disabled={salvandoPagamento} onClick={handleRegistrarPagamento}>{salvandoPagamento ? <Spinner /> : "Salvar"}</Btn>`

**Loading state:** `salvandoPagamento` boolean — replaces "Salvar" label with `<Spinner />` (reuse existing Spinner component, unchanged)

**State reset on cancel:** clear form fields, set `registrandoPagamento = false`

**State reset on success:** clear form fields, set `registrandoPagamento = false`, reload `pagamentosContrato` list

### 3c — "Pagamentos Recebidos" Collapsible Section

**Toggle header (always visible):**
```jsx
<div
  onClick={() => setPagamentosAberto(p => !p)}
  style={{
    display: "flex",
    alignItems: "center",
    justifyContent: "space-between",
    cursor: "pointer",
    padding: "8px 0",
    marginTop: 0,
  }}
>
  <p style={{
    fontFamily: "'Space Grotesk',sans-serif",
    fontWeight: 700,
    fontSize: 13,
    color: "#0f172a",
    margin: 0,
  }}>
    Pagamentos Recebidos {pagamentosAberto ? "▲" : "▼"}
  </p>
</div>
```

Starts collapsed: `const [pagamentosAberto, setPagamentosAberto] = useState(false)`

Load trigger: `useEffect(() => { if (!pagamentosAberto || pagamentosCarregado) return; ... }, [pagamentosAberto, contrato.id])`

**Content card (visible when `pagamentosAberto === true`):**
```jsx
<div style={{
  background: "#fff",
  borderRadius: 12,
  padding: "16px 20px",
  border: "1px solid #e2e8f0",
  marginTop: 8,
}}>
```
Matches Histórico section card exactly (border #e2e8f0, borderRadius 12, padding 16px 20px).

**Loading state:**
```jsx
<p style={{ textAlign: "center", fontSize: 13, color: "#94a3b8", padding: "16px 0" }}>
  Carregando pagamentos...
</p>
```

**Empty state (when list loaded and length === 0):**
```jsx
<div style={{ textAlign: "center", padding: "24px 0", color: "#94a3b8" }}>
  <p style={{ fontWeight: 700, fontSize: 13, marginBottom: 4 }}>Nenhum pagamento registrado.</p>
</div>
```
No secondary body line — single heading sufficient for this empty state.

**Payment list table (when list length > 0):**

Table container: `<div style={{ overflowX: "auto" }}>`
Table: `<table style={{ width: "100%", borderCollapse: "collapse", fontSize: 12 }}>`

Column headers (th style object — reuse existing `th` const):

| Column | Header label | Width hint |
|--------|-------------|------------|
| Data | DATA | ~80px |
| Valor | VALOR | ~100px |
| Parcelas | PARCELAS AMORTIZADAS | flexible |
| Observação | OBSERVAÇÃO | flexible |
| Ação | (blank) | ~40px |

**Payment row (td style object — reuse existing `td` const):**
- Row separator: `borderBottom: "1px solid #f1f5f9"` — matches parcelas table pattern
- No hover/click — rows are not navigable

Each row renders:
1. **Data cell:** `fmtData(pagamento.data)` — e.g. "15/04/2026"
2. **Valor cell:** `fmtBRL(pagamento.valor)` — e.g. "R$ 1.500,00"
3. **Parcelas cell:** inline chip list — see spec below
4. **Observação cell:** `pagamento.observacao || "—"` — truncate at 60 chars with "..."
5. **Ação cell:** delete button — see spec below

**Parcelas chip list (Claude's Discretion — inline text, not styled chips):**
- Render as plain text: `"Parc. 1/12, 2/12, 3/12"`
- fontSize: 12, color: #374151 — matches td body text
- No separate badge styling — keeps row visually clean
- If list is empty (should not happen in practice): "—"

**Delete button [X]:**
```jsx
<button
  onClick={() => handleExcluirPagamento(pagamento)}
  style={{
    background: "none",
    border: "none",
    cursor: "pointer",
    fontSize: 13,
    color: "#dc2626",
    fontWeight: 700,
    padding: "0 4px",
  }}
  title="Excluir pagamento"
>
  [X]
</button>
```
Uses text symbol `[X]` — consistent with existing destructive pattern in codebase. Color #dc2626 for destructive affordance without a styled confirm modal.

## Interaction Contract

### Registrar Pagamento — Full Flow

| Step | Trigger | Visual Change | State Change |
|------|---------|---------------|-------------|
| 1 | Click "Registrar Pagamento" | Button hidden; form card appears with border #c7d2fe | `registrandoPagamento = true` |
| 2 | User fills Data, Valor, Observação | — | local form state updates |
| 3 | Click "Salvar" | "Salvar" label → `<Spinner />`, Cancelar disabled | `salvandoPagamento = true` |
| 4a | Validation fails (valor ≤ 0) | toast.error("Informe um valor maior que zero.") | `salvandoPagamento = false` |
| 4b | Validation fails (valor > saldo) | toast.error("Valor superior ao saldo devedor (R$ X,XX).") — X formatted via fmtBRL | `salvandoPagamento = false` |
| 4c | Validation fails (data > hoje) | toast.error("Data não pode ser futura.") | `salvandoPagamento = false` |
| 5 | SP success | toast.success("Pagamento registrado. N parcela(s) amortizada(s).") — N from SP return | form cleared, `registrandoPagamento = false`, `salvandoPagamento = false`, pagamentos list reloaded |
| 6 | SP error (network/DB) | toast.error(err.message) | `salvandoPagamento = false` — form stays open |
| 7 | Click "Cancelar" | Form card hidden; button reappears | `registrandoPagamento = false`, form fields cleared |

Saldo validation (D-04): Calculate before SP call. Sum `calcularSaldoPorDividaIndividual` for each parcela where `saldo_quitado === false`. Parcelas available from `dividas` prop already in DetalheContrato scope.

### Excluir Pagamento — Full Flow

| Step | Trigger | Visual Change | State Change |
|------|---------|---------------|-------------|
| 1 | Click [X] on payment row | `window.confirm("Excluir este pagamento vai reverter a amortização das parcelas. Confirmar?")` appears | — |
| 2a | User clicks Cancel in confirm | Nothing changes | — |
| 2b | User clicks OK in confirm | [X] button area shows `<Spinner />` (or row grayed) | `excluindoPagamentoId = pagamento.id` |
| 3 | SP `reverter_pagamento_contrato` success | Toast success; row removed from list | `excluindoPagamentoId = null`, list reloaded |
| 4 | SP error | toast.error(err.message) | `excluindoPagamentoId = null` |

Loading state for delete: set `excluindoPagamentoId` to the payment id being deleted. Render `<Spinner />` in place of [X] button for that specific row. Other rows remain interactive.

Toast após exclusão: `toast.success("Pagamento excluído. Amortização revertida.")`

### Collapsible Toggle — "Pagamentos Recebidos"

- Initial state: collapsed (`pagamentosAberto = false`)
- First expand: triggers lazy load via useEffect
- Subsequent expand/collapse: no refetch (data already in state) — unless `pagamentosCarregado = false` (e.g. after a successful register or delete, force reload by resetting `pagamentosCarregado = false`)
- Toggle header always clickable, even during loading

## TIPO_EVENTO_LABELS Addition (HIS-05)

The existing `TIPO_EVENTO_LABELS` constant in DetalheContrato.jsx must be extended with two new entries:

```js
pagamento_recebido:  "Pagamento recebido",
pagamento_revertido: "Pagamento revertido",
```

Timeline dot style for these events: same as other non-criacao events — `background: "#4f46e5"`, `border: "2px solid #4f46e5"`.

Snapshot fields to render for `pagamento_recebido` event in Histórico:
- valor: formatted with fmtBRL
- parcelas_amortizadas: array of parcela numbers, rendered as "Parc. 1/12, 2/12"

For `pagamento_revertido` event:
- Same fields as pagamento_recebido (snapshot of what was reversed)

## Design Decisions — Claude's Discretion

The following items from CONTEXT.md "Claude's Discretion" are resolved here:

| Item | Decision | Rationale |
|------|----------|-----------|
| Field order in form | Data first, Valor second, Observação full-width | Date entry is the primary unique identifier; value follows naturally |
| Parcelas display style | Plain text "Parc. 1/12, 2/12" in td — no badge chips | Rows may have many parcelas; chips would overflow; text is scannable |
| Loading state on Salvar | `<Spinner />` replaces button label — same as Phase 6 edit pattern | Consistent with established Spinner usage |
| Empty state of Pagamentos Recebidos | Single line "Nenhum pagamento registrado." no secondary copy | Sufficient — user just registered from form above; no next-step hint needed |
| Total pago acumulado no header | NOT included — section stays minimal | Section header is a toggle; adding total adds complexity for marginal benefit; financial summary card already shows "Total Quitado" |
