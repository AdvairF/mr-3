<domain>
## Phase Boundary

Refatorar o módulo Fila para puxar **CONTRATOS** (`contratos_dividas`) ao invés de **DEVEDORES**, com ciclo de status/atendimento independente por contrato. 1 contrato = 1 linha. Devedor com N contratos = N linhas (repetição aceita por D-pre-5).

Histórico de atendimentos legado vinculado a `devedor_id` é descartado conscientemente (D-pre-7 / γ). Atendimentos novos passam a ser por contrato.

**Não muda:** módulos fora da Fila (Pessoas listing, Detalhe-Devedor com lembretes/registros_contato, Dashboard cards de lembretes) — preservados conforme D-pre-11.

</domain>

<decisions>
## Implementation Decisions

### Estrutura (LOCKED no briefing pré-discuss)

- **D-pre-1:** Fila puxa CONTRATOS (`contratos_dividas`), não devedores.
- **D-pre-2:** Cada contrato vira 1 linha (devedor com 2 contratos = 2 linhas).
- **D-pre-3:** Por linha mostra: nome devedor + credor + saldo atualizado + clicar abre `DetalheContrato` (componente existente, sem mudança).
- **D-pre-4:** Status/atendimento PER CONTRATO. Cada contrato tem ciclo próprio: `em_cobranca`, `em_localizacao`, `em_negociacao`, `notificado`, `ajuizado`, `quitado`, `arquivado`. Independente entre contratos do mesmo devedor.
- **D-pre-5:** Telefone/CPF do devedor pode aparecer 2x (B1) — sem complexidade extra.
- **D-pre-6:** Todos os contratos aparecem (incluindo quitados/arquivados) com toggle UI default ON para esconder. Filtros de status existentes preservados.
- **D-pre-7:** DROP legado de `eventos_andamento` (γ). Banco minúsculo (5 devedores ativos). Histórico legado descartado como decisão de produto. Atendimentos novos vão pro contrato.

### Schema (decisões Q1-Q2 + adições)

- **D-pre-10 (Migration manual em Supabase SQL Editor — robusta a 2 strategies):**
  ```sql
  -- Bloco 1: status por contrato
  ALTER TABLE public.contratos_dividas
    ADD COLUMN IF NOT EXISTS status TEXT NOT NULL DEFAULT 'em_cobranca'
    CHECK (status IN ('em_cobranca','em_localizacao','em_negociacao','notificado','ajuizado','quitado','arquivado'));

  -- Bloco 2: contrato_id em eventos
  ALTER TABLE public.eventos_andamento
    ADD COLUMN IF NOT EXISTS contrato_id UUID REFERENCES public.contratos_dividas(id);

  -- Bloco 2.5: PRE-FLIGHT — confirma acesso à tabela antes do drop legado
  SELECT COUNT(*) FROM public.eventos_andamento;

  -- Bloco 3: γ — DROP histórico legado (decisão de produto, banco minúsculo)
  -- STRATEGY A (preferida): TRUNCATE — fast, reseta sequences, requer privilégio TRUNCATE
  TRUNCATE public.eventos_andamento;

  -- STRATEGY B (fallback): se TRUNCATE der erro de permissão (RLS bypass / privilégio),
  -- substituir Bloco 3 por:
  --   DELETE FROM public.eventos_andamento;
  -- DELETE respeita RLS e não requer privilégio TRUNCATE. Mesmo efeito final no contexto γ
  -- (banco minúsculo, < 1000 rows). Documentar no SUMMARY qual strategy foi aplicada.

  -- Bloco 4: NOT NULL após drop limpo
  ALTER TABLE public.eventos_andamento
    ALTER COLUMN contrato_id SET NOT NULL;

  -- Bloco 5: PostgREST schema reload
  SELECT pg_notify('pgrst', 'reload schema');
  ```
  Risco γ aceito: histórico de eventos perdido. Decisão de produto registrada em SUMMARY.
  Migration robusta a 2 strategies (TRUNCATE preferida, DELETE fallback).

### UI Fila (Q5-Q7)

- **Cards do FilaPainel (Q5)** — recalcular por contrato:
  - **Pendentes** = contratos sem evento_andamento hoje
  - **Atendidos Hoje** = contratos com evento_andamento hoje
  - **Eventos Hoje** = `COUNT(eventos_andamento WHERE data_evento >= hoje)` — global, sem mudança
  - **Valor em Aberto** = `SUM(saldo_atualizado_por_contrato)`
- **Toggle "Esconder quitados/arquivados" (Q6)** — default ON. Filtro client-side `status NOT IN ('quitado','arquivado')`. Toggle desativa filtro.
- **Busca nome/CPF (Q7)** — filtra contratos cujo `devedor.nome` ou `devedor.cpf_cnpj` match. 1 devedor com 2 contratos = 2 linhas retornadas.
- **Filtros existentes preservados** — Status (com novos valores D-pre-4), Credor, Prioridade, valor min/max.
- **Ordenação intra-tab "Pendentes"** — preserva ordenação atual: ALTA→MEDIA→BAIXA, depois saldo desc, depois mais antigo.
- **Componente clicável** — clique na linha abre `DetalheContrato` (componente existente em `components/DetalheContrato.jsx` — sem mudança).
