    ## Lessons

    1. **Bugs dormentes só descobertos em audit pré-rewrite** — `filaDevedor.js`

       desde Phase 5 quando schema migrou para `contratos_dividas`). Funções nunca
       foram chamadas em prod (Fila não usava entrarNaFila/reciclarContratos
       automaticamente). Rewrite total absorveu fix-forward — sem necessidade de
       isolar 7.13c. Lição transferível: sempre incluir grep global de stale refs
       como gate pre-execute em rewrite total (`feedback_grep_global_before_
       pointwise_fix.md` aplicada).

    2. **Migration γ-aware em banco minúsculo** — D-pre-7 DROP histórico aceitável
       quando: (a) banco minúsculo (<1000 rows); (b) decisão de produto explícita;
       (c) STRATEGY A/B robusta cobre erro de privilégio; (d) audit pós-migração
       valida shape antes de rewrite. Aplicado: pre-flight SELECT COUNT, TRUNCATE
       preferida com DELETE fallback, NOT NULL após drop limpo, pg_notify reload.

    3. **Reuse de adapter D-02 evita helper proliferation** — investigação prévia
       (D-pre-12) confirmou que `calcularDetalheEncargosContrato` (Phase 7.8 D-02)
       já provê `.saldoAtualizado` por contrato. Wrapper thin de 1-linha
       suficiente para reduzir ruído nos callsites. Pattern: ANTES de criar
       helper novo, grep da função alvo para confirmar que adapter contrato-level
       não existe. Lição transferível: helpers de adapter são reutilizáveis —
       resistir ao impulso de criar helper novo quando adapter existente já cobre.

    4. **Aliases legacy como ponte de transição** — Plan 01 manteve aliases
       (listarDevedoresParaFila/proximoDevedor/alterarStatusDevedor) para back-
       compat com FilaDevedor.jsx atual; Plan 02 removeu APÓS rewrite UI consumir
       só nomes novos. Pattern: rewrites em sequência (service → UI) ganham
       atomicidade quando service expõe ambos nomes durante transição. Custo:
       1 task adicional (Plan 02 Task 7) para limpeza.

    5. **Cross-entity UAT obrigatório (lição feedback_cross_entity_uat_isolation
       .md cumulative)** — UAT-2 (devedor 1 com 2 contratos vs devedor 2 com 1

       7.8 (Mendes/TRADIO contaminação) ensinou: features com state global SEMPRE
       precisam cenário cross-entity como UAT obrigatório.

    ## Patterns

    - **Migration manual SQL Editor robusta a 2 strategies** — TRUNCATE preferida
      (fast, reseta sequences); DELETE fallback se TRUNCATE der erro RLS/privilégio.
      Pattern transferível.
    - **Pre-flight SELECT COUNT antes de DROP** — Bloco 2.5 da migration confirma
      acesso à tabela antes do drop irreversível. Pattern transferível.
    - **NOT NULL após drop limpo** — adicionar coluna NULLABLE primeiro, drop
      legado, depois SET NOT NULL. Evita FK violation na inserção de rows
      transitórios.
    - **PostgREST schema reload** — pg_notify('pgrst','reload schema') ao final
      da migration. Sem isso, PostgREST cache antigo serve queries que não
      reconhecem coluna nova.
    - **Wrapper thin compondo adapter existente** — calcularSaldoContratoAtualizado
