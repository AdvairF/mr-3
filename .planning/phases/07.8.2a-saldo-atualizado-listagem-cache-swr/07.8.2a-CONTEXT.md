## 1. Problema

Listagem de contratos (`ModuloContratos.jsx`) mostra só valores NOMINAIS: `valor_total`, `num_documentos`, `num_parcelas`, `em_atraso`. Advogado não vê saldo ATUALIZADO (Art. 354 CC) na visão agregada — tem que clicar em cada contrato pra ver o valor real. Com 200 contratos, é inviável.

Phase 7.8 trouxe saldo atualizado no DetalheContrato (4ª coluna + modal). Phase 7.8.2a leva pra listagem via cache SWR (padrão Stripe/Shopify/Vercel) — instantâneo ao voltar pra tela, revalida em background.

## 2. Escopo aprovado

**Fix A — Hook `useSaldoAtualizadoCache` (novo em `src/hooks/`)**
- Singleton `Map<contratoId, { saldo, detalhe, fingerprint, status, calculadoEm }>`
- Pub/sub com `Set<listener>` — componente se inscreve por contrato.id
- Fingerprint **deep** (12 campos motor-relevantes — ver D-06)
- API: `useSaldoContrato(contrato, dividas, pagamentos, hoje)`, `invalidateContrato(id)`, `invalidateAll()`, `removeContrato(id)`, `kickoffBatch(contratos, dividas, pagamentos, hoje)`
- `window.focus` listener no module-load (guard `typeof window`)
- Retorna 3 estados: `"loading"` (miss), `"fresh"` (fingerprint match), `"revalidating"` (stale ou drift)

**Fix B — `CelulaSaldoAtualizado.jsx` (novo em `src/components/`)**
- Consome `useSaldoContrato`
- 3 estados com fade 180ms: skeleton dots (loading), valor pleno (fresh), valor opacity 0.75 + title tooltip (revalidating)
- **Display-only** (não clicável) — row inteira continua navegando pro DetalheContrato

**Fix C — `batchCalculator.js` (novo em `src/utils/`)**
- `runBatches(items, batchSize, processBatch)` — `requestIdleCallback` com fallback `setTimeout(0)`, timeout 500ms

**Fix D — `ModuloContratos.jsx` (modificado)**
- Nova coluna "Saldo Atualizado" (header + `<CelulaSaldoAtualizado />`)
- Header ganha botão "🔄 Atualizar" (invalidateAll + kickoffBatch) + timestamp "Atualizado às HH:MM" (lê max calculadoEm via subscribe)
- `kickoffBatch` no mount com deps `[contratos.length, dividas.length, allPagamentosDivida.length]`
- Guard StrictMode via `useRef` flag `kickedOff.current`

**Fix E — 6 callers mutation (D-05 enforcement)**
- `DetalheContrato.jsx::handleRegistrarPagamento` → `invalidateContrato(contrato.id)` pós `toast.success`
- `DetalheContrato.jsx::handleExcluirPagamento` → idem
- `DetalheContrato.jsx::handleSalvarParcelasCustom` → idem
- `DetalheContrato.jsx::handleExcluirDocumento` → idem
- `DetalheContrato.jsx::handleExcluirContrato` → `removeContrato(contrato.id)` (contrato deixou de existir)
- `AdicionarDocumento.jsx::handleSalvar` → `invalidateContrato(contrato.id)` pós sucesso

**Fix F — Testes shield (3 novos em `src/services/__tests__/saldoAtualizadoCache.test.js`)**
- Shield 1: cache cross-entity (mudança em B não afeta A)
- Shield 2: fingerprint muda com virada de data Goiânia
- Shield 3: `invalidateContrato(A)` não afeta B/C

    const D = dividasDoContrato.map(d => [
      d.id, d.valor_total, d.data_vencimento, d.data_origem,
      d.data_inicio_atualizacao, d.indexador || "", d.multa_pct || 0,
      d.juros_tipo || "", d.juros_am || 0, d.honorarios_pct || 0,
      d.art523_opcao || "", d.saldo_quitado ? 1 : 0,
    ].join(":")).sort().join("|");
    const P = pagamentosFiltered.map(p =>
      [p.id, p.divida_id, p.valor, p.data_pagamento].join(":")
    ).sort().join("|");
    // Phase 7.9 amend — custas summary (global, belt: detecta mudança em count/total
    // de custas por divida) + _so_custas flag sequence (suspenders: detecta flip da
    // flag sem count/total mudar). Ambos defendendo D-05 handlers (criarCusta,
    // editarCusta, excluirCusta, togglePagoCusta) contra silent-stale.
    const C = JSON.stringify(
      dividasDoContrato
        .map(d => [d.id, (d.custas || []).length, (d.custas || []).reduce((s, c) => s + Number(c.valor || 0), 0)])
        .sort((a, b) => String(a[0]).localeCompare(String(b[0])))
    );
    const S = dividasDoContrato
      .map(d => [d.id, d._so_custas ? 1 : 0])
      .sort((a, b) => String(a[0]).localeCompare(String(b[0])))

      .join("|");
    return `${dataHoje}|D:${D}|P:${P}|C:${C}|S:${S}`;

- **D-07 (SWR semantic — 3 conceitos distintos)**:
  - Cache key = `contrato.id`
  - Fingerprint = campos do motor (detecta props drift)
  - Revalidação = triggers 1-5 (força recompute regardless de fingerprint)
  - Hash NÃO decide revalidar — decide cache hit/miss na leitura.

- **D-08 (5 gatilhos)**:
  1. Mount cache vazio → `kickoffBatch` em batches de 10 via `requestIdleCallback`
  2. Mount cache preenchido → render instantâneo stale + revalida background
  3. `window.focus` → `invalidateAll()` (multi-usuário)
  4. Evento pagamento/documento → `invalidateContrato(id)` ou `removeContrato(id)` (D-05)
  5. Virada do dia Goiânia → `dataHoje` no fingerprint muda → drift automático

- **D-09 (Data Goiânia)** — `Intl.DateTimeFormat('sv-SE', {timeZone:'America/Sao_Paulo'}).format(new Date())` retorna "YYYY-MM-DD" fuso correto.
