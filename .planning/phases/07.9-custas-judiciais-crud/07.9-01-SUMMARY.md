### Auto-fixed Issues

**1. [Rule 1 - Bug] dbGet API signature mismatch**
- **Found during:** Task 3
- **Issue:** PLAN's snippet usava sintaxe de objeto `dbGet("dividas", { id: 'eq.${dividaId}', limit: 1 })`, mas a assinatura real é `dbGet(t, queryString)` (PostgREST query string).
- **Fix:** Trocado para `dbGet("dividas", \`id=eq.${encodeURIComponent(dividaId)}&limit=1\`)` em editarCusta/excluirCusta/togglePagoCusta.
- **Files modified:** src/services/contratos.js
- **Commit:** c60c38a

**2. [Rule 1 - Bug] Btn não suporta prop `type` (causaria submit em botão Cancelar)**
- **Found during:** Task 2
- **Issue:** PLAN's snippet usava `<form onSubmit>` + `Btn type="button"` / `type="submit"`, mas o componente `Btn` não pass-through `type`. Cancelar dentro de form nativo dispararia submit.
- **Fix:** Substituído `<form>` por `<div>` simples, com `onClick={handleSalvarClick}` direto no Btn Salvar (sem submit semantics).
- **Files modified:** src/components/NovaCustaModal.jsx
- **Commit:** c60c38a

**3. [Rule 1 - Bug] Shield 19 — uppercase indexador codes não casam com motor**
- **Found during:** Task 6 (regression run)
- **Issue:** PLAN's snippet usava `indexador: "IPCA"` / `"IGPM"` (uppercase) e dívida `_so_custas:true`. Motor (lines 484-485 devedorCalc.js) usa hardcoded `"inpc"` para `_so_custas:true`, ignorando `divida.indexador` — então IPCA e IGPM produziam o MESMO valor (falha do test). Adicionalmente, `calcularFatorCorrecao` usa keys lowercase ("ipca", "igpm") em `getIndicesMerged()` — uppercase retornava fator=1.
- **Fix:** Trocou fixture para dívida regular (`_so_custas:false`) onde motor usa `divida.indexador !== "nenhum" ? indexador : "inpc"` (line 439). Trocou indexador para lowercase `"ipca"` / `"igpm"`. Documentado no comentário da Shield 19 explicando a divisão entre _so_custas:true (hardcoded "inpc") e regular dividas (D-25 herança).
- **Files modified:** src/services/__tests__/custasCache.test.js
- **Commit:** c60c38a
