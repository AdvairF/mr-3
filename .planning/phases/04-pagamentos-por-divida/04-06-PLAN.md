# CR-05 (Opção D): Persistir saldo_atual no banco — BACKLOG

## Status

**BACKLOG — não executado.** Depende de migration Supabase. Planejado para v1.2 ou após Phase 5.

## Diagnóstico

`TabelaDividas` calcula saldo via `calcularSaldosPorDivida(devedorCalc.js)` que lê
`pagamentos_parciais`. Dívidas com pagamentos em `pagamentos_divida` mostram saldo errado
na lista global. O fix atual (CR-05 Opção C, plan 04-05 + TabelaDividas) resolve dívidas
com `saldo_quitado=TRUE` (exibe 0), mas dívidas **parcialmente pagas** ainda exibem saldo
de `pagamentos_parciais` — potencialmente errado.

## Solução (Opção D)

Persistir o saldo correto no banco no mesmo PATCH que já atualiza `saldo_quitado`.
`recalcularESincronizar` já computa `novoSaldo` (Art. 354 CC sobre `pagamentos_divida`).
Basta salvá-lo em `dividas.saldo_atual` e `TabelaDividas` ler direto da row.

## Migration SQL necessária

```sql
-- Executar no Supabase SQL Editor antes de deployar o código
ALTER TABLE dividas ADD COLUMN IF NOT EXISTS saldo_atual NUMERIC;
```

## Tasks

### Task 1 — Migration Supabase
Executar o ALTER TABLE acima no SQL Editor do projeto Supabase.
Verificar: `SELECT column_name FROM information_schema.columns WHERE table_name='dividas' AND column_name='saldo_atual';`

### Task 2 — atualizarSaldoQuitado persiste saldo_atual (dividas.js)

Localizar `atualizarSaldoQuitado`. Adicionar parâmetro `novoSaldo` e incluir no PATCH:

```js
// ANTES:
export async function atualizarSaldoQuitado(dividaUuid, quitado, statusAtual) {
  const novoStatus = ...;
  return sb(..., "PATCH", { saldo_quitado: quitado, status: novoStatus, updated_at: ... });
}

// DEPOIS:
export async function atualizarSaldoQuitado(dividaUuid, quitado, statusAtual, novoSaldo) {
  const novoStatus = ...;
  return sb(..., "PATCH", {
    saldo_quitado: quitado,
    status: novoStatus,
    ...(novoSaldo !== undefined && { saldo_atual: novoSaldo }),
    updated_at: ...
  });
}
```

### Task 3 — recalcularESincronizar passa novoSaldo (PagamentosDivida.jsx)

```js
// ANTES:
await atualizarSaldoQuitado(divida.id, quitado, statusAtual);

// DEPOIS:
await atualizarSaldoQuitado(divida.id, quitado, statusAtual, novoSaldo);
```

### Task 4 — TabelaDividas lê d.saldo_atual com fallback (TabelaDividas.jsx)

```js
// ANTES:
const saldoExibido = d.saldo_quitado === true ? 0 : saldo;

// DEPOIS:
const saldoExibido = d.saldo_atual !== null && d.saldo_atual !== undefined
  ? d.saldo_atual
  : (d.saldo_quitado === true ? 0 : saldo);
```

### Task 5 — Verificação

## Restrições

- Não tocar `devedorCalc.js`
- `saldo_atual` é nullable — rows antigas (sem pagamento via nova tabela) ficam NULL e o fallback garante compatibilidade
- Não remover o atalho `saldo_quitado === true ? 0` — mantê-lo como segunda camada de segurança
