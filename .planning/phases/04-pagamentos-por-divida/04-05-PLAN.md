<objective>
Corrigir bug de sincronização identificado no UAT (CR-04):

Dois campos de quitação conviviam sem sincronização:
- `dividas.saldo_quitado` — atualizado automaticamente após cada operação de pagamento
- `dividas.status` — nunca atualizado automaticamente; permanecia "em cobrança" mesmo com saldo zerado

Resultado visível:
- Coluna "Atraso" em TabelaDividas exibia badge "Saldo quitado" (lendo `saldo_quitado`)
- Coluna "Status" exibia "Em Cobrança" (lendo `status`)
- Filtro "Status: Quitada" não encontrava dívidas com saldo_quitado=TRUE

Fix: extender `atualizarSaldoQuitado` para incluir `status` no mesmo PATCH, com salvaguarda
que preserva valores manuais como "acordo". Os 4 call sites de `recalcularESincronizar`
passam `divida.status` como parâmetro para que a salvaguarda funcione corretamente.

NÃO tocar: `devedorCalc.js`, `pagamentos_parciais`, nenhuma tabela DB além do PATCH existente.
Fix é em 2 arquivos — ~10 linhas no total.
</objective>

<salvaguarda>
A lógica da salvaguarda na reversão:
- quitado=TRUE  → novoStatus = "quitada"   (sempre — intencional)
- quitado=FALSE e statusAtual="quitada"    → novoStatus = "em cobrança" (reversão segura)
- quitado=FALSE e statusAtual="acordo"     → novoStatus = "acordo"      (preservado)
- quitado=FALSE e statusAtual=qualquer outro → novoStatus = statusAtual (preservado)

Nota: divida.status é lido do prop no momento da chamada. Como não há UI ativa que altere
dividas.status manualmente no codebase atual, o valor do prop reflete o estado real do banco
para os cenários de uso existentes.
</salvaguarda>
