## Lições transferíveis (5 + 2 novas)

### Lição 1 — Plan grep audit precisa cobrir state-pattern paralelo (NOVA)

**Sintoma:** Plan original (Task 3 critério Inp.jsx consumers) listou apenas componentes que fazem `import { Inp }` ou `import DiretrizesContrato`. NÃO buscou:
- `useState({.*indexador|multa_pct|juros_tipo}` (state pattern paralelo)
- `?? "igpm"|"fixo_1"|"2"|"10"` (defaults velhos como fallback em outros componentes)
- Helpers standalone como `initEditForm()` (DetalheContrato.jsx L75-91 escapou)

**Lição:** Antes de declarar varredura completa para mudanças de pattern (defaults, fallbacks, validation), grep amplo por:
1. Importadores diretos (consumers do componente shared)
2. State patterns equivalentes em OUTROS componentes (mesmo shape, sem o componente shared)
3. Defaults literais (especialmente `?? "<value>"` ou `useState({...})`)
4. Helper functions standalone que reproduzem o pattern (não-óbvias via IMPORT grep)

**Memory feedback NOVO:** `grep_state_pattern_paralelo_em_componentes_relacionados`

**Aplicação:** Quando phase muda DEFAULTS/FALLBACKS, usar 4-grep audit (não 1-grep IMPORT).

### Lição 4 — Motor lê das DIVIDAS, não do contrato header

**Descoberta arquitetural confirmada:**
- `contratos_dividas` (header) tem encargos declarativos
- `dividas` (rows individuais) também tem encargos
- **Motor jurídico (`devedorCalc.js`) usa `dividas`** para cálculos saldo/juros/multa/honor

**Implicação:** Corrupção visual de `contratos_dividas.multa_percentual` (bug SC4b) **NÃO afeta cálculos jurídicos**. Motor preserva valores corretos via dividas.

**Mitigação real:** ACORDO dividas (12 rows) tinham multa=10/juros=1/honor=10 corretos durante todo o bug. Restauração SQL do header foi cosmética.

## Patterns reutilizáveis

### Pattern 1 — `camposCriticosOk` validação client-side com condicional

```js
const camposCriticosOk =
  !!encargos.indexador &&
  !!encargos.multa_pct &&
  !!encargos.juros_tipo &&
  !!encargos.honorarios_pct &&
  (encargos.juros_tipo !== "outros" || !!encargos.juros_am);
```

Aplicado em 3 componentes (NovoContrato + AdicionarDocumento + DetalheContrato). Plain `const` em arquivos pequenos (~170-227 linhas), `useMemo` em arquivo grande (DetalheContrato 1326 linhas).

### Pattern 2 — Defesa em profundidade: botão disabled + handleSalvar early-return

```jsx
<span title={!camposCriticosOk ? "Preencha todos os encargos" : undefined}>
  <Btn disabled={salvando || !camposCriticosOk} onClick={handleSalvar}>
    Salvar
  </Btn>
</span>
```
+
```js
async function handleSalvar() {
  if (!camposCriticosOk) {
    toast.error("Preencha todos os encargos antes de salvar.");
    return;
  }
  // ... payload ...
}
```

**Defesa em profundidade:** botão disabled é defesa primária, early-return é fallback caso DOM seja manipulado via DevTools.

### Pattern 3 — Sentinel "— Selecione —" disabled bw-compat

Em correcao.js (consts shared):
```js
{ v:"", l:"— Selecione —", disabled:true }
```

Em Inp.jsx (shared component, bw-compat):
```jsx
<option disabled={typeof o === "object" && o.disabled === true}>
```

**Bw-compat short-circuit:** options legadas sem `disabled` key resolvem `false` → comportamento idêntico ao pré-fix. Validado em 5 outros consumers (App, DetalheContrato, DividaForm, NovaCustaModal, DiretrizesContrato).
