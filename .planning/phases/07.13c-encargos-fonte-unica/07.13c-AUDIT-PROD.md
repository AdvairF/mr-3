## Q1 — Drift contratos_dividas vs dividas (escopo do bug)

### SQL

```sql
SELECT
  cd.id AS contrato_id,
  cd.referencia,
  cd.multa_percentual      AS contrato_multa,
  cd.juros_am_percentual   AS contrato_juros,
  cd.honorarios_percentual AS contrato_honor,
  cd.indice_correcao       AS contrato_indice,
  cd.art523_opcao          AS contrato_523,
  d.id                     AS divida_id,
  d.multa_percentual       AS divida_multa,
  d.juros_am_percentual    AS divida_juros,
  d.honorarios_percentual  AS divida_honor,
  d.indice_correcao        AS divida_indice,
  d.art523_opcao           AS divida_523
FROM contratos_dividas cd
JOIN dividas d ON d.contrato_id = cd.id
WHERE
  cd.multa_percentual         IS DISTINCT FROM d.multa_percentual
  OR cd.juros_am_percentual   IS DISTINCT FROM d.juros_am_percentual
  OR cd.honorarios_percentual IS DISTINCT FROM d.honorarios_percentual
  OR cd.indice_correcao       IS DISTINCT FROM d.indice_correcao
  OR cd.art523_opcao          IS DISTINCT FROM d.art523_opcao
  OR cd.juros_tipo            IS DISTINCT FROM d.juros_tipo
  OR cd.data_inicio_atualizacao IS DISTINCT FROM d.data_inicio_atualizacao
ORDER BY cd.id, d.created_at;
```

### Resultado (orchestrator)

- **3 contratos com drift** (entre eles M L FRIOS `335a2ad2` confirmado)
- **147 dívidas afetadas** (parcelas dos 3 contratos)
- **100% incidência** no fluxo Fila / Carteira Total / DetalheContrato (todos 5 consumers do adapter herdam o drift)

**Conclusão:** drift é **sistêmico** dentro dos contratos afetados, não anedotal. Bug latente desde Phase 7.8 (introdução do `calcularDetalheEncargosContrato`).

## Q2 — Heterogeneidade intra-contrato

### SQL

```sql
SELECT
  contrato_id,
  COUNT(DISTINCT multa_percentual)      AS multas_distintas,
  COUNT(DISTINCT juros_am_percentual)   AS juros_distintos,
  COUNT(DISTINCT honorarios_percentual) AS honor_distintos,
  COUNT(DISTINCT indice_correcao)       AS indices_distintos,
  COUNT(DISTINCT art523_opcao)          AS art523_distintos,
  COUNT(*) AS total_dividas
FROM dividas
WHERE contrato_id IS NOT NULL
GROUP BY contrato_id
HAVING COUNT(DISTINCT multa_percentual) > 1
    OR COUNT(DISTINCT juros_am_percentual) > 1
    OR COUNT(DISTINCT honorarios_percentual) > 1
    OR COUNT(DISTINCT indice_correcao) > 1
    OR COUNT(DISTINCT art523_opcao) > 1;
```

### Resultado (orchestrator)

- **0 rows** — encargos são **uniformes intra-contrato** em todos os contratos com `contrato_id IS NOT NULL`
- Dentro de cada contrato com drift, todas as 49 (média) dívidas têm os mesmos valores entre si — só divergem do header

**Conclusão:** heterogeneidade intra-contrato é regra de zero ocorrência. **Path A puro é seguro arquiteturalmente** — não existe caso de uso em prod onde dividas individuais legitimamente discordem entre si.

**Implicação para D-pre-5 (Q5 do briefing):** `art523_opcao` pode ser tratado per-CONTRATO sem perda — não foi observada heterogeneidade nesta coluna.
