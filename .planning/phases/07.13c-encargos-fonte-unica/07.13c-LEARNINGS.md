## Lições transferíveis

### 1. ABORT GATE funcionou — pegou inversão arquitetural antes de tocar código

ABORT GATE D-pre-7.3 do Plan 01 NEW (UAT humano com 3 decisões PASS/PASS+ação/ABORT) capturou divergência entre premissa do orchestrator e modelo mental real do produto. Plan 01 NEW (SQL puro fail-fast antes de refactor) posicionou ABORT GATE no momento certo — antes de Plan 02 NEW tocar código.

Pattern transferível: phases com decisão arquitetural profunda devem ter validation gate fail-fast antes de implementação. Custo do gate (Plan 01 NEW: ~30 min de SQL + COMPARISON.md) << custo de implementação errada (Plan 02 + Plan 03 + revert + reciclagem).

### 2. Audit SQL ≠ entendimento de domínio

Q1+Q2 audits (3 contratos / 147 dividas / 0 heterogeneidade intra-contrato) estatisticamente corretos. Mas interpretação ("drift acidental") era assumption do orchestrator não validada com usuário antes de discuss-phase formal.

Pattern transferível: dados estatísticos não substituem entendimento de modelo mental do produto pelo dono. Discovery pré-discuss deve incluir pergunta direta ao usuário: "esse comportamento é por design ou é bug?" NÃO assumir interpretação técnica.

### 3. UAT humano em Task 3 CHECKPOINT é detector primário

Plan 01 NEW Task 3 CHECKPOINT estruturou 3 decisões explícitas (PASS/PASS+ação/ABORT) com perguntas direcionadas ao usuário sobre magnitude do delta E sobre realidade do negócio. Foi a pergunta sobre "realidade jurídica" que disparou ABORT.

Pattern transferível: CHECKPOINTs humanos devem incluir perguntas que questionam ASSUMPTIONS, não apenas validem NÚMEROS. "Esse delta é o que você esperava?" detecta ABORT por surpresa quantitativa. "Isso reflete a realidade jurídica/operacional?" detecta ABORT por surpresa qualitativa.

### 4. Premissa POST-HOC não substitui CONTEXT.md inteiro pelo dono

D-pre-14 da Phase 7.13b foi inserida como POST-HOC durante UAT 7.13b PAUSA #1-RETOMADA. CONTEXT.md da 7.13c herdou essa premissa sem sessão própria de discuss-phase com o dono. Resultado: 5 commits + 1644 linhas de plans baseados em assumption não validada.

Pattern transferível: D-pre POST-HOC herdadas em phase filha exigem validação fresca com dono ANTES de discuss-phase formal. Não tratar como "premissa locked" sem questionamento.
