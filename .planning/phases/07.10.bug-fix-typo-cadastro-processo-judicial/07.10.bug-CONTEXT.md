## Causa raiz (confirmada via investigação 2026-04-27)

**Arquivo:** `src/mr-3/mr-cobrancas/src/services/processosJudiciais.js`
**Funções afetadas:** `criar(dados)` (L40) e `adicionarDevedor({...})` (L62)

```js
// supabase.js:16 — assinatura sb()
export async function sb(path, method = "GET", body = null, extra = "") {
  const res = await fetch(`${SUPABASE_URL}/rest/v1/${path}${extra}`, ...)
//                                                          ^^^^^^^ template literal coerce-to-string
```

```js
// processosJudiciais.js:40 (BUG)
return sb(TABLE, "POST", {...}, { headers: { Prefer: "return=representation" } });
//                              ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ vira "[object Object]"
```

JS faz `String({headers:...})` → `"[object Object]"` no template literal `${path}${extra}` → URL final tem `processos_judiciais[object Object]` colado, PostgREST falha schema cache lookup.

**Garantia de equivalência pós-fix:** o header `Prefer: "return=representation"` JÁ ESTÁ HARDCODED em `sb()` linha 25. Passar de novo via 4º arg é **redundante E bugado**. Remover apenas elimina o bug — comportamento HTTP idêntico.

## Decisões locked (D-pre)

### D-pre-1 — Fix em 2 linhas (L40 + L62 processosJudiciais.js): remoção de 4º arg redundante

**Why:** 4º argumento `{ headers: { Prefer: "return=representation" } }` é redundante (header já hardcoded em `sb()` linha 25) E bugado (objeto coerce-to-string em template literal). Remoção elimina bug sem alterar comportamento HTTP.

**How to apply:**
- L40 (em `criar()`): trocar `}, { headers: { Prefer: "return=representation" } });` por `});`
- L62 (em `adicionarDevedor()`): mesma mudança
- 2 deletions, zero additions, zero refactor
