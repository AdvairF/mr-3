<objective>
CR-06: Adicionar campos de encargos (Diretrizes do Contrato) ao formulário de criação de
contratos, propagar esses encargos para cada parcela gerada, e exibir os encargos no DetalheContrato.

Problema resolvido: contratos criados na Fase 5 nasciam com encargos zerados hardcoded
(indice_correcao:"igpm", juros:0, multa:0). A correção monetária e os encargos só podiam
ser configurados editando cada parcela individualmente — fluxo manual e inconsistente com
o NovaDivida.jsx que já tem toda essa seção.

Saída: DiretrizesContrato.jsx (componente extraído de DividaForm), DividaForm.jsx refatorado
para usá-lo, NovoContrato.jsx com encargos, contratos.js com propagação via spread,
DetalheContrato com exibição read-only, Migration 3 SQL.
</objective>

Decisões de design para este plano:

- D-09: Nomes de coluna em contratos_dividas DEVEM coincidir com dividas para propagação
  por spread sem mapeamento: indice_correcao (não "indice"), juros_tipo (não "taxa_juros_preset"),
  art523_opcao (não "art_523_cpc"), custas (não "custas_adicionais")

- D-10: data_inicio_atualizacao no contrato = template. null → cada parcela usa sua própria
  data_vencimento (comportamento original de 05-01 preservado). Não nulo → todas as parcelas
  herdam o mesmo valor.

- D-11: custas no contrato é template JSONB copiado para cada parcela. Representa encargos
  judiciais base do contrato — o advogado pode depois editar por parcela individual.

- D-12: Nomes de campo no form seguem convenção de DividaForm (indexador, multa_pct, juros_am,
  honorarios_pct) para reutilizar DiretrizesContrato sem adaptações. O mapeamento form→banco
  ocorre apenas no handleSalvar de NovoContrato.

CRÍTICO: DividaForm.jsx é usado por NovaDivida.jsx — a refatoração para usar DiretrizesContrato
deve preservar comportamento 100% idêntico. Não alterar props, nomes de campo, valores default
nem estrutura visual de DividaForm.
</context>

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|

| T-05-11 | Tampering | custas JSONB spread para parcelas | mitigate | custas vem do form controlado (array de objetos com campos fixos); sem renderização de HTML, sem eval |
