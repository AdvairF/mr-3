## Interaction Contracts

### ModuloContratos вҖ” State Machine

```
view = "lista"         вҶ’ shows TabelaContratos + "Novo Contrato" button in header
view = "novo"          вҶ’ shows NovoContrato form (contrato header only)
view = "detalhe"       вҶ’ shows DetalheContrato (header + documentos list + "+ Adicionar Documento")
view = "parcela-detalhe" вҶ’ shows DetalheDivida for a parcela (navigated from inside DetalheContrato)
```

Navigation rules:
- "novo" back button вҶ’ "lista"
- "detalhe" back button вҶ’ "lista"
- Clicking a parcela row in DetalheContrato вҶ’ "parcela-detalhe" (sets `selectedParcela`)
- "parcela-detalhe" back button вҶ’ "detalhe" (returns to DetalheContrato, not "lista")
- `вҶҗ Ver documento` in DetalheDivida вҶ’ re-opens DetalheContrato scrolled to the documento section (implementation: call `onVerContrato(contrato_id)` which sets view="detalhe")
- `вҶҗ Ver contrato` in DetalheDivida вҶ’ same as above (both breadcrumbs land on DetalheContrato because DetalheDocumento is inline вҖ” not a separate view)

### TabelaContratos вҖ” Row Interactions

- Click anywhere on row: opens DetalheContrato
- Hover: `background: "#f8fafc"` on row, `cursor: "pointer"`, transition `.15s`
- Pagination: 20 rows per page, same Anterior/PrГіximo pattern as TabelaDividas
- Default sort: `created_at DESC` (most recent first)

Columns:

| # | Header | Width | Content |
|---|--------|-------|---------|
| 1 | Credor | auto | `credor.nome` or `вҖ” sem credor` in muted italic |
| 2 | Devedor | auto | `devedor.nome` вҖ” bold, color `#0f172a` |
| 3 | Docs | auto | `{num_documentos}` вҖ” count of documentos; `вҖ”` if 0 |
| 4 | Parcelas | auto | `{num_parcelas_total}x` вҖ” total parcelas across all docs |
| 5 | Valor Total | auto | `fmtBRL(valor_total)` вҖ” sum of doc values |
| 6 | Em Atraso | auto | count of parcelas where `saldo_quitado = false` and `data_vencimento < hoje`; `{n} parcelas` or `вҖ”` if zero; shown in `#dc2626` bold when > 0 |

Note: No "Tipo" column at contrato level вҖ” tipo is per Documento. TabelaContratos omits tipo column entirely.

### NovoContrato вҖ” Form Layout

This form creates ONLY the Contrato header. No tipo, no valor, no parcelas вҖ” those belong to Documentos.

Single card (`background: "#fff"`, `borderRadius: 16`, `padding: "20px 20px"`, `border: "1px solid #e8f0f7"`).

Field order (vertical stack, `gap: 12`):

1. **Credor** вҖ” `<select>` populated from `credores`. Optional. First option: `вҖ” Sem credor`.
2. **Devedor** вҖ” typeahead search input (same pattern as NovaDivida вҖ” min 2 chars, dropdown 8 results). Required.
3. **ReferГӘncia** вҖ” `<input type="text" placeholder="Ex.: Compra de maquinГЎrio Empresa X (opcional)">`. Optional. This is the human-readable name of the contract relationship.
4. **Encargos PadrГЈo** вҖ” block rendered by `DiretrizesContrato` component (already exists). Label above block: "Encargos PadrГЈo do Contrato" at 13px 700 Space Grotesk. Sub-label below: "Documentos herdam estes encargos вҖ” vocГӘ pode editar por documento" at 12px italic `#64748b`.

No `tipo`, `valor_total`, `data_inicio`, `num_parcelas`, or `primeira_parcela_na_data_base` fields in NovoContrato. These moved to `AdicionarDocumento`.

Field label style: `fontSize: 11`, `fontWeight: 700`, `color: "#64748b"`, `textTransform: "uppercase"`, `letterSpacing: ".05em"`, `display: "block"`, `marginBottom: 4`.

Input style: `width: "100%"`, `padding: "8px 12px"`, `border: "1.5px solid #e2e8f0"`, `borderRadius: 9`, `fontSize: 13`, `outline: "none"`, `fontFamily: "Plus Jakarta Sans, sans-serif"`, `boxSizing: "border-box"`.

Save/Cancel buttons at bottom:

```
[Criar Contrato]  [Cancelar criaГ§ГЈo]
```

`Criar Contrato` uses `color="#0d9488"`. Save button disabled until devedor is selected. On save: contrato is created, system navigates directly to `DetalheContrato` of the new contrato (view="detalhe") вҖ” not back to lista.

### AdicionarDocumento вҖ” Form Layout

Opened by clicking `+ Adicionar Documento` button in DetalheContrato. Renders BELOW the DetalheContrato header (not as a separate page вҖ” it slides in as an expanded section at bottom of DetalheContrato card). This keeps the 2-step flow (D-06) in a single view.

Implementation choice (Claude's discretion): inline form appended below the documentos list, toggled by a boolean state `adicionandoDocumento`. Back button hidden when form is open; form has its own `Cancelar` link.

Field order (vertical stack, `gap: 12`):

1. **Tipo** вҖ” `<select>` with 3 options: `NF/Duplicata`, `Compra e Venda`, `EmprГ©stimo`. Required. Default: `NF/Duplicata`.
2. **NГәmero do Documento** вҖ” `<input type="text" placeholder="Ex.: NF 001, NF 2024/123 (opcional)">`. Optional.
3. **Valor (R$)** вҖ” `<input type="number" min="0.01" step="0.01">`. Required.
4. **Data de EmissГЈo** вҖ” `<input type="date">`. Required.
5. **NВә de Parcelas** вҖ” `<input type="number" min="1" max="360">`. Required.
6. **Vencimento da 1ВӘ Parcela** вҖ” `<select>` with 2 options:
   - "Mesma data de emissГЈo" вҶ’ `primeira_parcela_na_data_base = true`
   - "Um mГӘs depois da emissГЈo" вҶ’ `primeira_parcela_na_data_base = false`
   Default: first option.
7. **Encargos do Documento** вҖ” block rendered by `DiretrizesContrato` component, pre-populated from contrato's encargos (inherited). Sub-label: "Herdados do contrato вҖ” edite se este documento tiver condiГ§Гөes diferentes" at 12px italic `#64748b`.

Preview section (when `valor` and `num_parcelas` are filled):

```
"{N} parcelas de R$ {base} (Гәltima: R$ {ultima})"
```

Style: `fontSize: 12`, `color: "#64748b"`, `fontStyle: "italic"`, `marginTop: 4`.

Save/Cancel buttons:

```
[Adicionar Documento]  [Cancelar]
```

`Adicionar Documento` uses `color="#0d9488"`. Disabled until tipo, valor, data_emissao, num_parcelas filled. On save: documento created + N parcelas generated + totais recalculated on contrato вҶ’ form hides вҶ’ documentos list refreshes.

### Loading and Error States

Loading (async ops): button label changes during save. No skeleton loaders вҖ” text placeholder sufficient.

- `NovoContrato` save button: `"Criando contrato..."` during save
- `AdicionarDocumento` save button: `"Adicionando documento..."` during save
