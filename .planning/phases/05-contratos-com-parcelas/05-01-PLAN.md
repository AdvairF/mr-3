<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Component → contratos.js | Component passes user-entered payload (tipo, credor_id, devedor_id, valor_total, num_parcelas, etc.) |
| contratos.js → Supabase REST | dbInsert/dbGet calls over authenticated fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | contratos_dividas table | mitigate | `CHECK (tipo IN ('NF/Duplicata','Compra e Venda','Empréstimo'))` in DDL enforces enum at DB level |
| T-05-02 | Tampering | dividas rows (parcela insert) | mitigate | RLS policy `auth.role() = 'authenticated'` on contratos_dividas + dividas already has RLS; parcelas inherit |
| T-05-03 | Elevation of Privilege | contratos_dividas (unauthenticated access) | mitigate | `ALTER TABLE contratos_dividas ENABLE ROW LEVEL SECURITY` + policy FOR ALL USING (auth.role()='authenticated') per D-01 |
| T-05-04 | Tampering | gerarPayloadParcelas (business logic bypass) | mitigate | num_parcelas CHECK (>= 1) at DB level; valor_total NUMERIC(15,2) NOT NULL; component validates before calling service |
| T-05-05 | Information Disclosure | contrato_id FK on dividas | accept | All dividas already protected by existing dividas RLS policy; adding nullable FK column does not widen attack surface |
</threat_model>

   ```js
   export async function criarContratoComParcelas(payload) {
     const res = await criarContrato(payload);
     const contrato = Array.isArray(res) ? res[0] : res;
     if (!contrato?.id) throw new Error("Supabase não retornou row do contrato");
     const parcelasPayload = gerarPayloadParcelas(contrato);
     const rows = [];
     for (const p of parcelasPayload) {
       const r = await dbInsert("dividas", p);
       rows.push(Array.isArray(r) ? r[0] : r);
     }
     return { contrato, parcelas: rows };
   }
   ```
